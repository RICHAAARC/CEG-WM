# 修复总结报告

## 执行摘要

本次修复地址了"静态代码审计即可确认差距项"中的 3 个关键问题：

### 问题 1：Workflow 入口一致性
**原问题**：`paper_faithfulness_spec.yaml`（规范文件）可被误传给 `paper_full_cuda` profile，导致运行期配置缺失关键字段。

**修复内容**：
- 文件：`scripts/run_onefile_workflow.py`
- 新增函数：
  - `_detect_cfg_role(cfg_obj: dict) -> str`：识别配置是规范（spec）还是运行期（runtime）
  - `_validate_cfg_role_for_profile(cfg_obj: dict, cfg_path: Path, profile: str) -> None`：验证配置角色与 profile 兼容性
- 在 main 函数中添加验证逻辑：在读取配置后立即检测角色并验证兼容性
- **效果**：当误传 spec 给 paper_full_cuda 时，直接失败并给出清晰错误信息，指向正确的 paper_full_cuda.yaml

**特征识别**：
- spec 配置特征：存在 `authority` 和 `audit_gate_requirements` 顶级字段
- runtime 配置特征：这些字段不存在

---

### 问题 2：Spec 与 Audit 清单不闭合
**原问题**：`paper_faithfulness_spec.yaml` 的 `audit_gate_requirements` 声明了 4 个必须的审计脚本，但这些脚本不存在，也未在 `run_all_audits.py` 的 `AUDIT_SCRIPTS` 列表中注册。

**修复内容**：
- 文件：`scripts/run_all_audits.py`
- 新增函数：
  - `_load_spec_audit_requirements(repo_root: Path) -> Dict[str, str]`：从 spec 加载审计需求
  - `_validate_spec_audit_closure(spec_audit_requirements: Dict, configured_scripts: List, repo_root: Path, strict: bool) -> List[Dict]`：验证所有声明的脚本都存在且已注册
- 在 main 函数中，所有脚本执行前调用闭合验证
- 路径规范化：统一处理 `scripts/audits/...` 和 `audits/...` 两种形式的路径

**执行机制**：
- 加载 spec 中的 `audit_gate_requirements` 字段
- 逐个检查声明的脚本文件是否存在+是否在 AUDIT_SCRIPTS 列表中
- 如果有缺失或未注册的脚本，在 --strict 模式下返回 FAIL 结果（severity=BLOCK）
- 在 AUDIT_SCRIPTS 列表中的脚本已包含：
  - `audit_paper_faithfulness.py`
  - `audit_paper_faithfulness_runtime_must_have.py`

**注**：spec 中声明的 4 个脚本目前仍不存在，该验证会正确报告闭合失败。解决路径是：
1. 创建这些脚本，或
2. 更新 spec，删除声明或指向已实现的脚本

---

### 问题 3：几何锚点必需性未纳入统一门禁
**原问题**：当 `geometry_gate_minimal` 被触发时，只强制 `geometry_evidence.sync.status`，但没有强制锚点字段（`anchor_digest`、`anchor_metrics`、`anchor_evidence_level`）。这导致锚点验证仅依赖 `assert_paper_mechanisms.py` 额外脚本，不够系统化。

**修复内容**：
- 文件：`configs/policy_path_semantics.yaml`
  - 更新 `geometry_gate_minimal` 字段，补齐 3 个锚点字段需求
  - 现在强制字段为：
    - `geometry_evidence.sync.status`
    - `geometry_evidence.anchor_digest` ✅ 新增
    - `geometry_evidence.anchor_metrics` ✅ 新增
    - `geometry_evidence.anchor_evidence_level` ✅ 新增

- 文件：`scripts/audits/audit_paper_faithfulness_runtime_must_have.py`
  - 新增检查函数：`check_geometry_anchor_completeness(repo_root: Path) -> Dict[str, Any]`
  - 在 `aggregate_checks()` 中调用此检查
  - 检查逻辑：
    - 仅当 geometry sync 成功时（status="ok"）强制锚点字段
    - 当 sync 失败时，锚点缺失被视为合法失败语义（不报错）
    - 提取 detect_record.json 进行验证

**字段来源**：
- 这些字段已在 `frozen_contracts.yaml` 的字段注册表中存在
- 修改仅为 append-only（补充到现有列表）
- 不破坏历史 records 语义
