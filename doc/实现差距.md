**主要差距（按优先级）**

1. `语义显著性` 目前还是“纹理代理”，不是真正语义显著性
- 当前 mask 算法是 `texture_gradient_v1`（梯度阈值+形态学），不是语义分割/语义显著图。见 `main/watermarking/content_chain/semantic_mask_provider.py:552`, `main/watermarking/content_chain/semantic_mask_provider.py:559`
- routing 也主要输出区域比例与二值映射摘要（`mask_true -> hf`, `mask_false -> lf`），更像“纹理分区路由”，不是“语义显著性驱动流形区域”。见 `main/watermarking/content_chain/semantic_mask_provider.py:705`, `main/watermarking/content_chain/semantic_mask_provider.py:711`


2. `自适应流形区域` 目前是比例级别自适应，不是空间/流形级自适应
- `SubspacePlanner` 会产出 `trajectory_jacobian_nullspace_svd` 的摘要，这点方向是对的。见 `main/watermarking/content_chain/subspace/subspace_planner_impl.py:251`
- 但 `build_band_spec_from_mask()` 只用 `mask area_ratio` 决定 `hf/lf region_ratio`，没有把语义 mask 的空间结构真正映射到可执行的局部流形区域。见 `main/watermarking/content_chain/subspace/subspace_planner_impl.py:1603`, `main/watermarking/content_chain/subspace/subspace_planner_impl.py:1608`, `main/watermarking/content_chain/subspace/subspace_planner_impl.py:1624`


3. 主嵌入路径仍以 `图像域 LF/HF` 为主，不是你预期的 `latent 扩散步注入` 主路径
- embed 主流程走的是 `content_real_v1` 图像管线，调用 `encode_low_freq_dct()` + `embed_high_freq_pattern()`。见 `main/watermarking/embed/orchestrator.py:557`, `main/watermarking/embed/orchestrator.py:604`, `main/watermarking/embed/orchestrator.py:631`
- LF 实现是 `block_dct_lf_modulation`（图像块 DCT 调制），不是 PRC 论文要求的 latent 符号翻转+BP 解码主实现。见 `main/watermarking/content_chain/low_freq_coder.py:841`, `main/watermarking/content_chain/low_freq_coder.py:948`
- HF 图像路径是 `texture_region_weak_pattern`，是最小可行实现，不是 T2SMark 级别 tail-truncation sampling。见 `main/watermarking/content_chain/high_freq_embedder.py:640`, `main/watermarking/content_chain/high_freq_embedder.py:721`


4. `latent 注入基础设施` 已有，但和 planner 输出还没完全打通
- 你已经有 SD3 callback 注入基础设施与 `LatentModifier`（这是很好的进展）。见 `main/diffusion/sd3/hooks.py:18`, `main/diffusion/sd3/hooks.py:34`, `main/watermarking/content_chain/latent_modifier.py:28`
- 但 `LatentModifier` 期待 `plan` 中有 `lf_basis/hf_basis`。见 `main/watermarking/content_chain/latent_modifier.py:134`, `main/watermarking/content_chain/latent_modifier.py:190`
- 当前 `SubspacePlannerImpl.plan()` 主要输出的是摘要/规格（`subspace_spec`, `high_freq_subspace_spec`, `basis_digest`），未见实际 `lf_basis/hf_basis` 在主 plan 输出中。见 `main/watermarking/content_chain/subspace/subspace_planner_impl.py:250`, `main/watermarking/content_chain/subspace/subspace_planner_impl.py:257`, `main/watermarking/content_chain/subspace/subspace_planner_impl.py:288`


5. 几何链模块齐全，但更像“代理几何证据”，还不是你描述的 attention-map 相对关系绑定
- `AttentionAnchorExtractor` 存在，但其关系摘要是从 latent token 向量相似度构造，不是直接从 Self-Attention map 关系提取。见 `main/watermarking/geometry_chain/attention_anchor_extractor.py:338`, `main/watermarking/geometry_chain/attention_anchor_extractor.py:353`, `main/watermarking/geometry_chain/attention_anchor_extractor.py:368`
