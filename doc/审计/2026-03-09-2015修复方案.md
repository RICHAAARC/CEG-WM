## 3. 修复方案整体思路

### 3.1 主链阻断层级

从目标字段反向设计：

```
[目标] embed: injection_mode = "subspace_projection", mask_digest ≠ null
    ↑
[需要] SemanticMaskProvider.extract() 返回 status="ok"
    ↑
[需要] _build_content_inputs_for_embed() 返回非 None（有 image_path）
    ↑
[需要] _resolve_embed_input_image_path() 返回非 None
    ↑
[阻断] STATIC-01: paper_full_cuda.yaml 无 embed.input_image_path
    ↑
[修复] P0-A: 在 run_embed.py 实现 Preview Generation，消除对外部图像依赖

[目标] detect: threshold_source = "np_canonical"
    ↑
[需要] get_np_threshold() 命中 cfg["__thresholds_artifact__"] 分支
    ↑
[需要] thresholds_artifact 在 detect 前注入 cfg
    ↑
[阻断] STATIC-03: run_detect.py 仅通过 thresholds_path 参数注入，scripts 不提供则 fallback
    ↑
[修复] P1-A: 在 paper_full_cuda.yaml 支持 detect.calibration_artifact_path，run_detect.py 中增加自动加载路径

[目标] calibrate/evaluate: 正确使用 NP 统计 (n_pos=100, n_neg=100)
    ↑
[需要] detect_records_glob 路径有效，正负样本 ≥100 对
    ↑
[阻断] STATIC-02: glob 为 null，seeds=[0,1] 产样不足
    ↑
[说明] P0-B: 本方案仅追加配置注释说明（seeds=[0,1] 与 glob=null 不变）
    STATIC-02 在本方案中归为**未修复**，需独立实施 seeds 扩展 + glob 注入步骤
```

### 3.2 优先修复路径

```
P0-A (Preview Generation) → 解决 STATIC-01，使链路 A 主链自足
P0-B (seeds 与 glob 配置) → 仅追加配置注释，STATIC-02 未修复
                            → 真正修复需：seeds 扩展为 range(100) + glob 注入步骤独立实施
P1-A (calibration_artifact_path) → 解决 STATIC-03，链路 C NP 判决自足
P1-B (FPR 改善建议) → 解决 STATIC-06，链路 D 统计达标
P2-A (LF BP 迭代) → 改善 STATIC-05，链路 B 图像域 sidecar lf_status 收敛改善
                  → 注意：仅影响图像域 sidecar 路径，不影响 detect_runtime_mode 门控（潜空间路径）
```

### 3.3 为什么这是最小风险方案

- P0-A 在现有推理调用路径内插入无注入推理，不改变任何字段语义、不新建写盘路径
- P1-A 仅在 detect 阶段读取一个可选配置字段，不改变 calibration 本身的执行逻辑
- P0-B 仅追加配置注释，不改变任何字段值或函数签名；STATIC-02 需独立步骤解决
- 所有修复均通过 append-only 字段扩展实现，不修改现有字段语义

#### 4.2 main/cli/run_embed.py 修改

**涉及函数：** `run_embed()`（主流程函数）

**修改目的：** 在 `_build_content_inputs_for_embed` 调用之前，若 `cfg["__embed_input_image_path__"]` 仍为 None，且 `embed.preview_generation.enabled == true`，则执行无注入 SD3 推理产生预览图，并将其路径写入 `cfg["__embed_input_image_path__"]`。

**插入位置：** 在现有代码中，`content_inputs_pre = embed_orchestrator._build_content_inputs_for_embed(cfg)` 这一行**之前**

**逻辑说明：**

**变量作用域说明：** `pipeline_obj` 和 `device` 作为局部变量，在 `run_embed.py` 的插入点约 160 行之后才被赋值；实现时须通过 `pipeline_result.get("pipeline_obj")` 和 `cfg.get("device", "cpu")` 替代，不可直接引用局部变量名。`ensure_output_layout` 之后存在第二个 `__embed_input_image_path__` 赋值块，因 CLI `input_image_path` 未提供且 `embed.input_image_path` YAML 字段不存在，该块不会覆盖 preview 临时路径。

```
1. 读取 cfg.get("embed", {}).get("preview_generation", {}).get("enabled", False)
2. 若 enabled=True 且 cfg.get("__embed_input_image_path__") 为 None：
   a. preview_pipeline_obj = pipeline_result.get("pipeline_obj")   # 通过 pipeline_result 取，插入点前已赋值
      preview_device = cfg.get("device", "cpu")                     # 从 cfg 读取，不依赖同名局部变量
      preview_seed = seed_value                                      # seed_value 已在插入点前赋值
      调用 infer_runtime.run_sd3_inference(
          cfg, preview_pipeline_obj, preview_device, preview_seed,
          injection_context=None,        # 无注入
          injection_modifier=None,        # 无修改器
          capture_final_latents=False     # 不需要 latents
      )
   b. 从 inference_result["output_image"] 获取预览图（PIL Image 对象）
   c. 使用 Python tempfile 将其保存为临时 PNG 文件（不写入 run_root，run_root 此时尚未创建）
   d. 将临时文件路径设为 cfg["__embed_input_image_path__"]
   e. 记录 preview 生成状态到 run_meta["preview_generation"] = {
          "enabled": True,
          "status": "ok" / "failed",
          "reason": ...(若 failed),
          "seed": preview_seed          # 记录 seed 用于对齐审计（对应 RISK-02）
      }
   f. 若 preview 推理失败（inference_status != "ok"），set run_meta["preview_generation"]["status"] = "failed"，
      不中断流程（保持 cfg["__embed_input_image_path__"] = None，使内容链以 status="failed" 返回，
      injection_mode 降级为 latent_direct_fallback，失败语义保持）
   ※ ensure_output_layout 之后存在第二个 __embed_input_image_path__ 赋值块；
     因 CLI input_image_path 未提供且 embed.input_image_path YAML 字段不存在，
     该块不会覆盖 preview 临时路径，preview 路径保留有效。
3. 继续原有 content_inputs_pre 构造流程
```

**输入变化：** `cfg["__embed_input_image_path__"]` 可能从 None 变为有效临时路径（当 preview generation 成功）
**输出变化：** `embed_record["preview_generation"]` 新增状态审计字段（append-only）

**失败语义保证：**
- preview 推理失败时**不中断流程**，失败语义传播为 `content_result.status = "failed"` → `injection_mode = "latent_direct_fallback"`
- 不将 failed 包装为 success，符合原则 6

**时序约束：**
- 此插入点在 layout 创建（`ensure_output_layout`）**之前**，因此 preview 图不能写入 artifacts/（run_root 尚未创建）
- 临时文件路径不属于 path_policy 约束的 run_root 写盘路径，合规
- preview artifact 在 layout 创建之后**可选**地将临时文件移至 `artifacts/preview/preview.png` 并记录 sha256（append-only）

**字段映射表（embed_record 新增字段）：**

| 字段路径 | 类型 | 描述 |
|---------|------|------|
| `embed_record.preview_generation.enabled` | bool | 是否尝试了 preview generation |
| `embed_record.preview_generation.status` | str | "ok" / "failed" / "skipped" |
| `embed_record.preview_generation.reason` | str \| null | failed 时的原因 |
| `embed_record.preview_generation.artifact_sha256` | str \| null | preview 图 sha256（若已保存到 artifacts）|

### P1-B：FPR 改善（解决 STATIC-06）

**优先级：** P1（统计目标未达成）
**影响范围：** 无直接代码修改（需要数据实验后决策）

**问题分析：**

- FPR = 0.02（2/100 负样本越过阈值）
- 根本原因：N=100 时校准分位精度为 ±1 样本（FPR 精度 ±1%），评估集与校准集 HF score 分布存在轻微偏移
- HF score 为 SD3 轨迹 L2 范数，对 watermarked/non-watermarked 分布重叠区过宽

**修复策略（分阶段）：**

**阶段一（可静态分析）：**
- 确认校准负样本与评估负样本来自不同 seed/prompt（分布独立性），排除采样偏差
- 确认 `exclude_synthetic_negative_closure_marker = true` 过滤逻辑正确覆盖了所有伪负样本

**阶段二（需真实 SD3 运行）：**
- 使用 N=200+ 校准样本（扩展校准集）以提高分位精度
- 分析 HF score 分布，调整 `watermark.hf.tau` 参数以增大 watermarked/non-watermarked 分布间距
- 若 HF score 分布问题根深，考虑加入 score normalization 步骤（归一化到 [0,1]），但须确认不改变 NP 判决规则语义

**修复约束：**
- 禁止在 evaluate 阶段重新估计阈值（原则 5）
- 调整仅在 calibration 阶段（扩大 N 或调整 HF 参数）
- 修改 HF 参数（tau 等）会改变 cfg_digest，需重新运行完整 embed/detect/calibrate/evaluate 链路

### P2-A：LF BP 迭代上限（改善 STATIC-05）
