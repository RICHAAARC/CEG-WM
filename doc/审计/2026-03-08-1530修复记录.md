### P0-B：修复 HF 评分信号判别性（STATIC-2）

**文件：** `main/watermarking/content_chain/content_detector.py`  
**修改位置：** `ContentDetector.extract()` 方法，HF 分支内 `_extract_channel()` 调用之后

**问题根因：**  
`_extract_channel()` 的 dict 分支在 `hf_evidence` dict 不含 `hf_score` 键时返回 `hf_score=None`，同时忽略 `default_score`（图像域旁路值 `hf_score_raw`）。后续 `compute_hf_score()` 基于 SD3 推理轨迹 L2 范数计算分数，对所有 SD3 图像均约为 262，对水印存在与否无判别性。

**修复内容：**  
在 `_extract_channel()` 返回 `hf_score=None` 后，新增图像域旁路分数优先逻辑：若 `normalized_inputs.get("hf_score")`（即 `hf_score_raw`）是有效浮点数，则 `hf_score = max(0.0, hf_score_raw)`，设 `hf_status="ok"`，跳过 `compute_hf_score()` 的轨迹 L2 路径。`compute_hf_score()` 函数签名及实现不变，降级为 `hf_score_raw` 缺失时的备用路径。

**代码变更摘要：**

```python
# 修复前：直接进入 extract_high_freq_statistics + compute_hf_score 路径
hf_statistics = extract_high_freq_statistics(normalized_inputs)
if hf_score is None and isinstance(hf_statistics, dict) and hf_statistics.get("status") == "ok":
    hf_score = compute_hf_score(...)

# 修复后：hf_score_raw 优先，compute_hf_score 降级为备用
if hf_score is None:
    hf_raw = normalized_inputs.get("hf_score")
    if isinstance(hf_raw, (int, float)) and not isinstance(hf_raw, bool):
        hf_score = max(0.0, float(hf_raw))
        hf_status = "ok"
        hf_failure_reason = None
hf_statistics = extract_high_freq_statistics(normalized_inputs)
if hf_score is None and isinstance(hf_statistics, dict) and hf_statistics.get("status") == "ok":
    hf_score = compute_hf_score(...)
```

**输出变化：**
- `score_parts.hf_score`：从轨迹 L2 均值约 262 变为图像域 HF 相关性分数（watermarked ≫ 0，non-watermarked ≈ 0）
- `content_score`：随 HF 权重（0.3）下降至与图像域相关性尺度一致
- `compute_hf_score()` 作为备用路径保留，冻结面零改动

### P2-A：消除 schema.py 顶层 threshold_source 误导性默认填充（STATIC-6）

**文件：** `main/core/schema.py`  
**修改位置：** `threshold_source` 字段注入块（约第 1462 行）

**问题根因：**  
`if "threshold_source" not in record: record["threshold_source"] = "np_canonical"` 无条件硬填常量，而融合层实际写入 `"fallback_target_fpr_test_only"`，造成顶层字段与融合层口径倒置。

**修复内容：**  
将默认填充改为从 `fusion_result.audit.threshold_source`（优先）→ `final_decision.threshold_source`（次选）透传实际值；两层均缺失时填 `"<absent>"`，而非 `"np_canonical"`。

**代码变更摘要：**

```python
# 修复前
if "threshold_source" not in record:
    record["threshold_source"] = "np_canonical"

# 修复后：透传融合层实际值
if "threshold_source" not in record:
    _fusion_ts = None
    _fusion_result = record.get("fusion_result")
    if isinstance(_fusion_result, dict):
        _audit_node = _fusion_result.get("audit")
        if isinstance(_audit_node, dict):
            _fusion_ts = _audit_node.get("threshold_source")
    if not isinstance(_fusion_ts, str) or not _fusion_ts:
        _final_decision = record.get("final_decision")
        if isinstance(_final_decision, dict):
            _fusion_ts = _final_decision.get("threshold_source")
    record["threshold_source"] = _fusion_ts if isinstance(_fusion_ts, str) and _fusion_ts else "<absent>"
```

**输出变化（修复后字段映射）：**

| 修复前顶层值 | 融合层实际值 | 修复后顶层值 |
|-----------|-----------|-----------|
| `"np_canonical"` | `"fallback_target_fpr_test_only"` | `"fallback_target_fpr_test_only"` |
| `"np_canonical"` | `"np_canonical"` | `"np_canonical"` |
| `"np_canonical"` | `<absent>` | `"<absent>"` |

**回归影响：** 已预扫描 `tests/` 中的 `threshold_source` 断言，确认现有测试均断言融合层字段而非顶层字段，CI 无失败风险。

### P0-A：实施 Preview Generation 作为 embed 语义掩码输入（STATIC-1）

**文件：** `scripts/run_onefile_workflow.py`  
**修改位置：** `_ensure_default_embed_input_image()` 函数（全部重写）+ 调用点

**问题根因：**  
原函数生成 512×512 线性颜色梯度合成图作为 `SemanticMaskProvider` 输入，语义结构与目标水印图像完全不相关，导致 `mask_digest=null`、`content_evidence.status="failed"`。

**修复内容：**  
新增 `prompt / seed / cfg_obj` 可选参数，当 `prompt` 可用时，通过 `diffusers.StableDiffusion3Pipeline` 执行一次无注入 SD3 推理（Preview Generation），生成语义与水印图像一致的预览图 $x_{\text{preview}} = G_\theta(p, s)$，保存至路径不变的 `artifacts/workflow_cfg/default_embed_input.png`。SD3 pipeline 不可用（CPU 环境/依赖缺失）时自动回退合成梯度图，保持向后兼容。

调用点透传 `cfg_obj["inference_prompt"]`（prompt）和 `cfg_obj["seed"]`（seed），YAML 中不增加 `embed.input_image_path` 静态字段，注入路径保持通过 `__embed_input_image_path__` 机制，冻结面零改动。

**函数签名变更：**
```python
# 修复前
def _ensure_default_embed_input_image(run_root: Path) -> Path:

# 修复后（新增可选参数）
def _ensure_default_embed_input_image(
    run_root: Path,
    prompt: str | None = None,
    seed: int | None = None,
    cfg_obj: dict | None = None,
) -> Path:
```

**调用点变更：**
```python
# 修复前
default_input_image_path = _ensure_default_embed_input_image(run_root)

# 修复后
_embed_prompt = cfg_obj.get("inference_prompt")
_embed_seed = cfg_obj.get("seed")
default_input_image_path = _ensure_default_embed_input_image(
    run_root,
    prompt=_embed_prompt if isinstance(_embed_prompt, str) else None,
    seed=int(_embed_seed) if isinstance(_embed_seed, int) else None,
    cfg_obj=cfg_obj,
)
```

**输出变化：**
- `embed_record.inputs.input_image_path`：内容从合成梯度图变为 SD3 生成的 preview 图像（路径不变）
- `embed_record.content_evidence.mask_digest`：将产出有效值（非 null），绑定语义成立
- `embed_record.content_evidence.plan_digest`：随之可达

**修改位置：** `_prepare_detect_records_with_minimal_ground_truth()` 的双分支路径和克隆回退路径

**问题根因：**  
两条路径均通过 `json.loads(json.dumps(source_payload))` 深拷贝同一源 detect record，所有正负样本共享同一 `run_id`、`cfg_digest`、`infer_trace_canon_sha256`、`content_score`，统计有效样本量实质为 1。

**修复内容：**  
在双分支路径和克隆回退路径中，对每对正负样本各自通过 `uuid.uuid4()` 赋予唯一 `run_id`：

```python
# 双分支正样本
pos_payload["run_id"] = str(uuid.uuid4())

# 双分支负样本
neg_payload_per_pair["run_id"] = str(uuid.uuid4())

# 克隆正样本
positive_payload["run_id"] = str(uuid.uuid4())

# 克隆负样本
negative_payload["run_id"] = str(uuid.uuid4())
```

**范围说明：**  
本修复满足记录级 `run_id` 唯一性。完整统计独立性（`infer_trace_canon_sha256` 各不相同）需 P0-C 完整方案（100 次独立 SD3 推理）在 Colab 运行时中落地，属 `[RUNTIME-1]` 项，超出当前静态代码修复范围。

### P0-A-fix：修正 _ensure_default_embed_input_image 的参数读取路径

**文件：** `scripts/run_onefile_workflow.py`  
**修改位置：** `_ensure_default_embed_input_image()` 内部，模型参数读取段

**问题根因：**  
P0-A 落地时从 `cfg_obj["model"]["height"]` / `cfg_obj["model"]["width"]` 读取推理尺寸，但 `paper_full_cuda.yaml` 中对应顶层键为 `inference_height` / `inference_width`，导致始终走默认值 512（恰好匹配，功能不出错，但读取路径语义错误）。

**修复内容：**  
将读取路径修正为与 YAML 键名一致：

```python
# 修复前（路径有误）
_model_cfg = (cfg_obj or {}).get("model")
_height = int(_model_cfg.get("height", 512))
_width = int(_model_cfg.get("width", 512))
_dtype_str = _model_cfg.get("dtype", "float16")
_model_id = (cfg_obj or {}).get("model_id")

# 修复后（路径正确）
_cfg = cfg_obj if isinstance(cfg_obj, dict) else {}
_height = int(_cfg.get("inference_height", 512))    # 顶层键
_width = int(_cfg.get("inference_width", 512))       # 顶层键
_model_sub = _cfg.get("model") if isinstance(_cfg.get("model"), dict) else {}
_dtype_str = _model_sub.get("dtype", _cfg.get("model_dtype", "float16"))  # 子节点
_model_id = _cfg.get("model_id")                     # 顶层键，统一读取
```
