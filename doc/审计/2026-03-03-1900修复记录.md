## 23. repro pipeline detect 阶段缺少 allow_threshold_fallback_for_tests override

### 23.1 问题描述（GPU run commit `274eb591`）

**触发错误**（来自 `compare_summary.json` stderr_tail）：
```
ValueError: np threshold artifact is required; set __thresholds_artifact__ or explicitly enable allow_threshold_fallback_for_tests
```

repro pipeline 执行顺序为 embed → detect → calibrate → evaluate。detect 阶段执行时 calibrate 尚未运行，故 `cfg["__thresholds_artifact__"]` 为 None。`get_np_threshold`（`main/watermarking/fusion/decision.py`）在无工件且 `allow_threshold_fallback_for_tests=False`（默认）时 fail-fast，导致 detect 进程 exit 1。

主流程（onefile workflow）的 detect 命令显式传递 `--override allow_threshold_fallback_for_tests=true`；`scripts/run_repro_pipeline.py` 的 `_build_stage_command` 未对 detect 阶段传递同等 override，属于行为不一致。

### 23.2 修复内容

**文件 1**：`scripts/run_repro_pipeline.py` → `_build_stage_command`

在 detect 阶段的 `command.extend(...)` 中追加 `allow_threshold_fallback_for_tests=true` override：

```python
# 变更前
if stage_name == "detect":
    command.extend([
        "--input",
        str(run_root / "records" / "embed_record.json"),
    ])

# 变更后
if stage_name == "detect":
    # repro 流水线：detect 在 calibrate 之前执行，尚无阈值工件，
    # 需显式启用 fallback，与 onefile 主工作流保持一致。
    command.extend([
        "--input",
        str(run_root / "records" / "embed_record.json"),
        "--override",
        "allow_threshold_fallback_for_tests=true",
    ])
```

`allow_threshold_fallback_for_tests` 在 `configs/runtime_whitelist.yaml`（行 230-231）已白名单化，可安全传入。

**文件 2**：`tests/test_repro_pipeline_detect_threshold_fallback_override.py`

新增 3 个回归测试（5 个 case）：
- `test_detect_stage_includes_fallback_override`：detect 命令包含该 override
- `test_detect_stage_override_is_properly_structured`：以 `--override key=value` 相邻参数对形式出现
- `test_non_detect_stages_do_not_include_fallback_override[embed/calibrate/evaluate]`：非 detect 阶段不携带该 override

全部 5 个 case 通过。
