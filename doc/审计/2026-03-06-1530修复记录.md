## 1. 执行范围

本次按 1530 修复方案落地如下条目：

1. `GAP-D1`：dual_branch 路径支持 `pair_count`，不再固定仅生成 1 对样本。
2. `GAP-B1`：LF PRC detect 支持 `numpy.ndarray` / tensor-like 扁平化，并补充维度审计字段。
3. `GAP-B1` 次级硬化：planner 仅保留 HF 通道 `max_cols=16` 截断，LF 通道不再截断。
4. `GAP-B2`：取消 `score_parts["lf_status"]` 被 PRC trace 无条件覆写，新增 append-only `prc_latent_status`。
5. detect 端验收口径对齐：`status=="ok"` 且 `mask_digest` 缺失时从 `input_record.content_evidence.mask_digest` 透传。
6. evaluate 端补齐 `exclude_synthetic_negative_closure_marker` 过滤逻辑，并在 `paper_full_cuda` 显式开启。
7. 补充最小新增回归测试，覆盖上述修复点。

## 2. 代码变更记录

### 2.1 `scripts/run_onefile_workflow.py`

变更点：

1. `_prepare_detect_records_with_minimal_ground_truth()` dual_branch 分支由固定单对样本改为按 `pair_count` 循环生成。
2. 命名兼容保持：
	- `pair_count == 1` 仍写 `detect_records_{stage}_gt_positive.json` / `detect_records_{stage}_gt_negative.json`
	- `pair_count > 1` 写入索引文件（`_000`、`_001` ...）
3. dual_branch 分支支持 `prompts` 注入到每个 pair。
4. clone fallback 路径中，当存在 `dual_branch_failure_reason` 时，negative 样本强制写入：
	- `calibration_sample_usage="synthetic_negative_for_ground_truth_closure"`
	- `dual_branch_failure_reason`

影响说明：

1. 修复 dual_branch 忽略 `pair_count` 的问题。
2. 保持字段兼容与现有审计字段语义。

### 2.2 `main/watermarking/content_chain/low_freq_coder.py`

变更点：

1. `LFCoderPRC.detect_score()` 在维度不足 `failed` trace 中新增：
	- `available_latent_dim`
	- `required_block_length`
2. `LFCoderPRC.detect_score()` `ok` trace 中追加同名维度审计字段。
3. `_flatten_recursive_to_list()` 补充：
	- `numpy.ndarray` 展平支持
	- tensor-like（具备 `detach/cpu/numpy`）对象展平支持（不新增 torch 强依赖）

影响说明：

1. detect 侧 latent 类型兼容增强，降低误报 `lf_insufficient_latent_dimension` 风险。
2. 维度不足时可直接通过 trace 字段定位。

### 2.3 `main/watermarking/content_chain/subspace/subspace_planner_impl.py`

变更点：

1. `_build_executable_basis_payload()` 改为仅 `channel == "hf"` 时执行 `max_cols=16` 截断。
2. `channel == "lf"` 不再强制截断列数。

影响说明：

1. LF 通道可保留更多维度，降低 PRC 解码可用性风险。
2. 未新增 planner fail-fast，保持最小改动原则。

### 2.4 `main/watermarking/detect/orchestrator.py`

变更点：

1. 在 `run_detect_orchestrator()` 收口分数后新增 `mask_digest` 透传调用。
2. `_bind_raw_scores_to_content_payload()`：
	- 保留 `score_parts["lf_detect_trace"]`
	- 不再覆写 `score_parts["lf_status"]`
	- 新增 append-only `score_parts["prc_latent_status"]`
3. 新增 `_populate_detect_mask_digest_from_input_record()`：
	- 当 `content_evidence_payload.status == "ok"` 且 `mask_digest` 缺失时，从 `input_record.content_evidence.mask_digest` 透传。
4. `_load_records_for_evaluate()`：
	- 读取 `evaluate.exclude_synthetic_negative_closure_marker`
	- 为 `True` 时过滤 `calibration_sample_usage == "synthetic_negative_for_ground_truth_closure"`
5. 新增 `_is_synthetic_negative_closure_sample()` 过滤辅助函数。

影响说明：

1. `lf_status` 与 `lf_score` 主语义不再被 PRC trace 后置冲掉。
2. detect 成功记录可补齐 `mask_digest` 验收锚点。
3. evaluate 阶段可与 calibration 一致过滤 synthetic closure 样本。

### 2.5 `configs/paper_full_cuda.yaml`

变更点：

1. `calibration.exclude_synthetic_negative_closure_marker: true`
2. `evaluate.exclude_synthetic_negative_closure_marker: true`

影响说明：

1. 在 paper_full 配置下显式启用 synthetic closure 样本过滤。
2. 默认行为（代码默认 `False`）未变，保持向后兼容。
