# 项目修复方案

> 审计版本号：2026-03-01-2330  
> 依据文档：`doc/审计/2026-03-01-2330审计报告.md`  
> 修订目标：在不引入新语义漂移的前提下，解除 workflow 阻断并恢复可解释统计闭环。

---

## 1. 修复目标与验收标准

### P0：解除内容链输入阻断（High-Critical）
- **目标**：Embed/Detect 的语义掩码提取均拿到有效主输入，消除 `mask_extraction_no_input` 主失败因。
- **验收**：
  - `embed_record.content_evidence.status == "ok"`
  - `detect_record.content_evidence_payload.status == "ok"`（至少不再因 `mask_extraction_no_input` 失败）
  - `embed_record.content_evidence.mask_digest` 非空。

### P1：恢复检测端 Real 模式判决（High）
- **目标**：消除长期 `fallback_identity_v0`，检测进入真实判决路径。
- **验收**：
  - `detect_record.detect_runtime_is_fallback == false`
  - `detect_record.detect_runtime_mode == "real"`
  - `detect_record.decision.is_watermarked` 非 null。

### P1：降低几何同步不确定性失效（High）
- **目标**：在不改全局默认语义的前提下，降低 `sync_uncertainty_too_high`。
- **验收**：
  - `geometry_failure_reason=sync_uncertainty_too_high` 占比显著下降（阶段门槛：<20%）。

### P1：防止校准样本语义污染（High）
- **目标**：禁止把结构性失败样本改写成 `status="ok"` 进入正式校准。
- **验收**：
  - `detect_record_for_calibration.json` 不再出现“原始 failed 被改写为 ok”且无来源标记的情况。
  - 校准输入可区分“真实可计算样本”与“流程补全样本”。

---

## 2. 变更执行计划（按风险最小化）

### 2.1 [Script] onefile 注入 Embed 输入路径（不改 paper 配置硬编码）
**文件**：`scripts/run_onefile_workflow.py`
- **变更动作**：
  - 在生成 runtime cfg 阶段，若 `embed.input_image_path` 缺失，则注入默认相对路径（如 `assets/test_images/...`）。
  - 注入前做存在性检查；不存在则 fail-fast，报可定位错误。
  - 注入结果写入 `artifacts/workflow_cfg/profile_paper_full_cuda.yaml`，用于可审计复盘。
- **不做事项**：
  - 不在 `configs/paper_full_cuda.yaml` 写死绝对路径。

### 2.2 [Code] 修复 Detect 内容链真实断点（含 planner 回退路径）
**文件**：`main/watermarking/detect/orchestrator.py`
- **变更动作**：
  - 新增 `_build_content_inputs_for_detect(cfg, input_record)`，复用 `_resolve_detect_image_path`。
  - 将首段 `content_extractor.extract(cfg)` 改为传入 `inputs=detect_inputs`。
  - 将 planner 回退分支 `planner_content_result = impl_set.content_extractor.extract(cfg_for_planner)` 同步改为传入 detect inputs，避免二次触发 `mask_extraction_no_input`。
- **说明**：
  - 仅补输入，不改 detect/content 的 frozen 语义枚举。

### 2.3 [Code] 保持 Embed 输入构造单一来源并增强可观测性
**文件**：`main/watermarking/embed/orchestrator.py`
- **变更动作**：
  - 保持 `_build_content_inputs_for_embed` 只从已解析的 `input_image_path` 构造输入。
  - 增加输入缺失时的审计日志字段（不改变返回结构），便于快速定位。
- **不做事项**：
  - 不新增多来源隐式 fallback，避免路径优先级歧义。

### 2.4 [Config/Override] 几何同步参数仅做 profile 级调优
**文件**：`scripts/run_onefile_workflow.py`（override）、可选 `configs/paper_full_cuda.yaml`
- **变更动作**：
  - 优先通过 paper profile override 设置 `embed.geometry.sync_strength`（建议先 `0.1`，必要时 `0.2`）。
  - 保持 `latent_sync_template.py` 中全局默认值 `0.01` 与阈值 `0.5` 不变。
- **理由**：
  - 避免改全局默认造成跨 profile 语义漂移与历史结果不可比。

### 2.5 [Script] 收紧 onefile 校准补全策略（禁止失败语义被“ok 化”）
**文件**：`scripts/run_onefile_workflow.py`
- **变更动作**：
  - `PAPER_SIDECAR_DISABLED_FALLBACK` 分支保留，但补全样本必须增加显式来源标记（如 `calibration_sample_origin: sidecar_disabled_fallback`）。
  - 正式校准输入默认仅接收真实 `status="ok"` 且可计算分数样本。
  - 若需流程连通，可将补全样本写入独立诊断文件，不直接进入正式 NP 校准集合。
- **不做事项**：
  - 不将 `status="failed"` 无条件改写为 `status="ok"`。

## 4. 风险控制与回滚

- 若 detect 输入接入改动导致新异常，优先回滚 `main/watermarking/detect/orchestrator.py` 的输入传递部分。
- 若 profile 级 `sync_strength` 调优导致指标反向退化，撤回 override，不修改全局默认。
- 若校准语义策略影响既有下游脚本，保留补全文件但从正式校准入口移除，保障主统计链纯度。

## 5. 本轮不承诺项（需 P2）

- 不承诺单样本条件下 `threshold_value != 0.0`。
- 不承诺当前阶段产出可发表的 `TPR@FPR` 结论。
- P2 需补齐样本量与 ground truth 后再做统计结论与消融结论。
