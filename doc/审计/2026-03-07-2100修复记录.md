### P1b：`DETECT_PREPLAN_INPUT` — detect pre-plan 传入图像路径

**文件：** `main/cli/run_detect.py`
**函数：** `run_detect()`（pre-plan 构造段）

**根因：** detect pre-plan 阶段調用 `content_extractor.extract(cfg_for_preplan)` 时 inputs=None，导致 SemanticMaskProvider 进入 `mask_extraction_no_input` 路径，`mask_digest=None`，subspace_planner 使用 fallback plan，detect-side `lf_basis` 无效。

**修改策略（最小侵入）：**
在 `bound_fact_sources` 上下文之外，以标准库 `json.load` 对 `input_record_path` 做最小解析，提取 `watermarked_path`，再传入 `extract(inputs={"image_path": ...})`。

优先级顺序：
1. `input_record_path` 中的 `watermarked_path`（验证路径可达性）
2. `cfg["detect"]["content"]["probe_image_path"]`（备用配置字段）
3. 无 inputs（维持原有行为，不阻断流程）

**变更 diff 摘要：**
```python
# 旧
content_result_pre = impl_set.content_extractor.extract(cfg_for_preplan)

# 新（含最小解析逻辑）
_preplan_inputs: dict | None = None
if input_record_path:
    try:
        with open(input_record_path, "r", encoding="utf-8") as _f:
            _pre_record = json.load(_f)
        _wm_path = _pre_record.get("watermarked_path")
        if isinstance(_wm_path, str) and _wm_path and _wm_path != "<absent>":
            _wm_path_resolved = str(Path(_wm_path).resolve())
            if Path(_wm_path_resolved).is_file():
                _preplan_inputs = {"image_path": _wm_path_resolved}
    except Exception as _pre_exc:
        print(f"[Detect][preplan] input_record 最小解析失败，使用无输入模式: {_pre_exc}")
if _preplan_inputs is None:
    _probe_img = cfg.get("detect", {}).get("content", {}).get("probe_image_path")
    if isinstance(_probe_img, str) and _probe_img:
        _preplan_inputs = {"image_path": _probe_img}
content_result_pre = impl_set.content_extractor.extract(cfg_for_preplan, inputs=_preplan_inputs)
```

**冻结面影响：** 无（不修改任何 records 字段名称、枚举、digest 口径）
**门禁面影响：** 无（不新增路由分支；在已有 `if input_record_path:` 条件内最小扩展）

**注意（RISK-008）：** `input_record_path` 在 pre-plan 和 `bound_fact_sources` 内分别被读取两次；两次读取间文件修改概率极低，无需额外处理。

## 六、BUG-FIX-1：`INJECTION_TRACE_DIGEST_MISMATCH` — 移除跨 run 注入 digest 等值门禁

**触发事件：** 本次修复后首次 Colab workflow 运行（"No Image Outputs" 输出集），embed 成功（`mask_digest` 非 null、图像已保存），detect 返回码 0 但 `detect_runtime_mode: fallback_identity_v0`、`content_evidence_payload.status: mismatch`、`decision_status: error`。

**文件：** `main/watermarking/detect/orchestrator.py`  
**函数：** `_evaluate_injection_consistency()`（第 1499 行）

**根因：**  
`_evaluate_injection_consistency` 对 embed 与 detect 的 `injection_trace_digest`、`injection_params_digest`、`subspace_binding_digest` 做相等性比较。但这三个 digest 均依赖各自推理步骤中的实际 latent 值：

- `injection_trace_digest`：每步注入修改向量的哈希，随 latent 轨迹变化
- `injection_params_digest`：注入参数（含实时 LF/HF 编码结果），随 latent 轨迹变化
- `subspace_binding_digest`：子空间绑定摘要，包含推理上下文相关量

embed 和 detect 使用不同推理种子，产生不同 latent 轨迹，因此三个 digest 跨 run 必然不等。比较结果恒为 `mismatch`，`forced_mismatch = True`，整个 detect scoring block 被跳过，`detect_runtime_mode` 永远不能设为 `"real"`。

**证据（来自 "No Image Outputs" detect_record）：**

| 字段 | embed 值 | detect 值 |
|------|---------|----------|
| `injection_trace_digest` | `7da7ec1e...` | `7488df26...` |
| `injection_params_digest` | `946bc75e...` | `81037fab...` |
| `injection_site_digest` | `bea636992b...` | `bea636992b...` ✓（相同）|

`injection_site_digest` 相同（子空间位置确定性），三个 runtime digest 不同（latent 数据相关）。

**修复策略：**  
移除 `_evaluate_injection_consistency` 中的跨 run 等值校验逻辑（包括 `injection_trace_digest_mismatch`、`injection_params_digest_mismatch`、`injection_subspace_binding_digest_mismatch` 三个 return 分支），保留格式有效性检查但改为返回 `"absent"`（而非 `"mismatch"`），不阻断主流程。子空间计划一致性已由 `plan_digest == plan_digest_expected` 机制独立保证。

**变更 diff 摘要：**
```python
# 旧（3个 mismatch 返回分支均已移除）:
if embed_trace_digest != detect_trace_digest:
    return "mismatch", "injection_trace_digest_mismatch"
if embed_params_digest != detect_params_digest:
    return "mismatch", "injection_params_digest_mismatch"
if isinstance(embed_binding_digest, str) and embed_binding_digest:
    if not isinstance(detect_binding_digest, str) or detect_binding_digest != embed_binding_digest:
        return "mismatch", "injection_subspace_binding_digest_mismatch"

# 新（仅保留格式检查，格式无效降级为 absent 不阻断）:
if not isinstance(embed_trace_digest, str) or not isinstance(detect_trace_digest, str):
    return "absent", "injection_trace_digest_invalid"
if not isinstance(embed_params_digest, str) or not isinstance(detect_params_digest, str):
    return "absent", "injection_params_digest_invalid"
# 跨 run 等值校验已移除：trace/params/binding digest 均为 latent-dependent，不可跨 run 比对。
return "ok", None
```

**预期效果：**
- `forced_mismatch = False` → detect scoring block 正常执行
- `detect_lf_status` = 基于 final_latents 的真实 LF 分数计算结果
- `detect_runtime_mode = "real"`（若 `detect_lf_status == "ok"` 且 pipeline 真实运行）
- `decision_status` 不再为 `"error"`

**回归测试：** 741 passed（全量 tests/ 通过，无新增专项测试需求）

**保留的有效门禁（不受影响）：**
- `injection_status == "ok"` 双端状态检查（embed 和 detect 均须注入成功）
- `plan_digest == plan_digest_expected` 计划一致性检查
- `injection_site_digest`（写入 record 供审计，不做门控比较）
