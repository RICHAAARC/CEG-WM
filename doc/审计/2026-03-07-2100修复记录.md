## 六、BUG-FIX-1：`INJECTION_TRACE_DIGEST_MISMATCH` — 移除跨 run 注入 digest 等值门禁

**触发事件：** 本次修复后首次 Colab workflow 运行（"No Image Outputs" 输出集），embed 成功（`mask_digest` 非 null、图像已保存），detect 返回码 0 但 `detect_runtime_mode: fallback_identity_v0`、`content_evidence_payload.status: mismatch`、`decision_status: error`。

**文件：** `main/watermarking/detect/orchestrator.py`  
**函数：** `_evaluate_injection_consistency()`（第 1499 行）

**根因：**  
`_evaluate_injection_consistency` 对 embed 与 detect 的 `injection_trace_digest`、`injection_params_digest`、`subspace_binding_digest` 做相等性比较。但这三个 digest 均依赖各自推理步骤中的实际 latent 值：

- `injection_trace_digest`：每步注入修改向量的哈希，随 latent 轨迹变化
- `injection_params_digest`：注入参数（含实时 LF/HF 编码结果），随 latent 轨迹变化
- `subspace_binding_digest`：子空间绑定摘要，包含推理上下文相关量

embed 和 detect 使用不同推理种子，产生不同 latent 轨迹，因此三个 digest 跨 run 必然不等。比较结果恒为 `mismatch`，`forced_mismatch = True`，整个 detect scoring block 被跳过，`detect_runtime_mode` 永远不能设为 `"real"`。

**证据（来自 "No Image Outputs" detect_record）：**

| 字段 | embed 值 | detect 值 |
|------|---------|----------|
| `injection_trace_digest` | `7da7ec1e...` | `7488df26...` |
| `injection_params_digest` | `946bc75e...` | `81037fab...` |
| `injection_site_digest` | `bea636992b...` | `bea636992b...` ✓（相同）|

`injection_site_digest` 相同（子空间位置确定性），三个 runtime digest 不同（latent 数据相关）。

**修复策略：**  
移除 `_evaluate_injection_consistency` 中的跨 run 等值校验逻辑（包括 `injection_trace_digest_mismatch`、`injection_params_digest_mismatch`、`injection_subspace_binding_digest_mismatch` 三个 return 分支），保留格式有效性检查但改为返回 `"absent"`（而非 `"mismatch"`），不阻断主流程。子空间计划一致性已由 `plan_digest == plan_digest_expected` 机制独立保证。

**变更 diff 摘要：**
```python
# 旧（3个 mismatch 返回分支均已移除）:
if embed_trace_digest != detect_trace_digest:
    return "mismatch", "injection_trace_digest_mismatch"
if embed_params_digest != detect_params_digest:
    return "mismatch", "injection_params_digest_mismatch"
if isinstance(embed_binding_digest, str) and embed_binding_digest:
    if not isinstance(detect_binding_digest, str) or detect_binding_digest != embed_binding_digest:
        return "mismatch", "injection_subspace_binding_digest_mismatch"

# 新（仅保留格式检查，格式无效降级为 absent 不阻断）:
if not isinstance(embed_trace_digest, str) or not isinstance(detect_trace_digest, str):
    return "absent", "injection_trace_digest_invalid"
if not isinstance(embed_params_digest, str) or not isinstance(detect_params_digest, str):
    return "absent", "injection_params_digest_invalid"
# 跨 run 等值校验已移除：trace/params/binding digest 均为 latent-dependent，不可跨 run 比对。
return "ok", None
```

**预期效果：**
- `forced_mismatch = False` → detect scoring block 正常执行
- `detect_lf_status` = 基于 final_latents 的真实 LF 分数计算结果
- `detect_runtime_mode = "real"`（若 `detect_lf_status == "ok"` 且 pipeline 真实运行）
- `decision_status` 不再为 `"error"`

**回归测试：** 741 passed（全量 tests/ 通过，无新增专项测试需求）

**保留的有效门禁（不受影响）：**
- `injection_status == "ok"` 双端状态检查（embed 和 detect 均须注入成功）
- `plan_digest == plan_digest_expected` 计划一致性检查
- `injection_site_digest`（写入 record 供审计，不做门控比较）

## 八、BUG-FIX-2：`LF_CODEWORD_MISMATCH` — embed/detect 两侧码字不一致

**执行日期：** 2026-03-08  
**触发事件：** n=100 GPU run 结果，`lf_score = 0.5`（随机基线），`bp_iteration_count = 50`（全耗尽不收敛）。

**文件：**
- `main/watermarking/content_chain/channel_lf.py`
- `main/diffusion/sd3/infer_runtime.py`

**根因：**  
embed 侧 `apply_low_freq_encoding_torch()` 使用 `key = watermark_seed + step_index`（步骤相关种子）通过 `torch.randint` 随机生成 ±1 码字，28 个扩散步骤产生 28 个不同随机码字，累积信号方向互相抵消。detect 侧 `detect_score()` 使用 `canonical_sha256({plan_digest, "lf_message"})` 派生固定 LDPC 码字并计算 agreement，两侧码字完全独立，导致 `lf_score = (随机对齐比例) ≈ 0.5`。

**修复策略：**

1. `channel_lf.py` — 新增辅助函数族：
   - `_get_ldpc_codeword_bits(cfg, n)` — 与 `detect_score()` 使用**完全相同**的种子推导逻辑（`canonical_sha256({plan_digest, "lf_message"})` → `encode_message_bits()`），返回 96 位 ±1 码字列表
   - `_derive_ldpc_codeword_from_cfg(cfg, n, device)` — torch 张量版本
   - `_derive_ldpc_codeword_from_cfg_np(cfg, n)` — numpy 数组版本
   - `_has_plan_digest(cfg)` — 判断 cfg 中是否存在有效 `plan_digest`
   - 修改 `apply_low_freq_encoding_torch()` 和 `apply_low_freq_encoding()`：码字改用上述固定 LDPC 码字，伪高斯幅度部分仍用步骤相关 key（符号固定 + 幅度随机 → 28 步相干叠加）

2. `infer_runtime.py` — `_build_injection_cfg()` 末尾追加三个字段：
   - `"lf_plan_digest": context.plan_digest`
   - `"lf_message_length": lf_cfg.get("message_length", 64)`
   - `"lf_ecc_sparsity": lf_cfg.get("ecc_sparsity", 3)`

**码字一致性验证：**
```
embed codeword[:8]  = [-1, -1, 1, -1, 1, 1, 1, 1]
detect expected[:8] = [-1, -1, 1, -1, 1, 1, 1, 1]
全 96 位一致: True
```

**回归测试：** 741 passed（2 skipped）

## 九、BUG-FIX-3：`LF_PROJECTION_MISSING` — detect 侧未投影到 LF 子空间即解码

**执行日期：** 2026-03-08  
**触发事件：** 同 BUG-FIX-2（n=100 run，`lf_score = 0.5`）。

**文件：**
- `main/watermarking/content_chain/low_freq_coder.py`
- `main/watermarking/detect/orchestrator.py`

**根因：**  
embed 侧将 `lf_basis` 投影后的 96 个 LF 系数打入水印再重建回 latent 空间；detect 侧 `detect_score()` 直接对展平后的原始 latent 取前 96 个坐标做 LDPC 解码，完全跳过 `lf_basis` 投影。两者工作在不同空间，detect 读到的是原始 latent 坐标，而非已编码的 LF 系数，LDPC 解码无信号可恢复。

**修复策略：**

1. `low_freq_coder.py` — `detect_score()` 新增可选参数 `lf_basis: Optional[Dict[str, Any]] = None`：
   - 若提供 `lf_basis`，先调用 `channel_lf.compute_lf_basis_projection[_torch]()` 将 latent 投影到 LF 子空间，得到 96 个 LF 系数
   - 回退语义：若 `lf_basis = None` 或投影失败，降级为展平原始 latent（维持原有行为）
   - trace 中新增 `lf_projection_used: bool` 字段用于审计

2. `orchestrator.py` — `_extract_content_raw_scores_from_image()` 内 `lf_coder.detect_score()` 调用处：
   - 从 `plan_dict.get("lf_basis")` 提取 `lf_basis_for_decode`
   - 传入 `detect_score(..., lf_basis=lf_basis_for_decode)`

**变更 diff 摘要：**
```python
# orchestrator.py（修复 Bug-A）
lf_basis_for_decode = plan_dict.get("lf_basis") if isinstance(plan_dict.get("lf_basis"), dict) else None
lf_score, prc_trace = lf_coder.detect_score(
    cfg=cfg,
    latent_features=detect_latents,
    plan_digest=plan_digest,
    cfg_digest=cfg_digest,
    lf_basis=lf_basis_for_decode,   # 新增：投影到 LF 子空间
)

# low_freq_coder.py detect_score()（修复 Bug-A）
if lf_basis is not None and isinstance(lf_basis, dict):
    projected = channel_lf.compute_lf_basis_projection_torch(latent_features, lf_basis)
    flat_latents = projected.detach().cpu().numpy().reshape(-1).tolist()
    lf_projection_used = True
else:
    flat_latents = _flatten_to_list(latent_features)
```
