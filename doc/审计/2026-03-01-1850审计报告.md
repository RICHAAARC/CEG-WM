# 项目风险与研究差距综合审计报告

> 审计版本号：2026-03-01-1850
> 审计方式：静态源码审计 + 现有运行产物复核（`outputs/colab_complete_workflow/colab_run_paper_full_cuda`）  
> 审计目标：判断当前实现与“语义显著性自适应流形水印 + 内容/几何双证据链 + NP 阈值融合”研究目标的差距，并区分可静态确认项与需真实 SD3.5 workflow 验证项。

---

## 一、最终结论（可对外复核）

当前项目**具备论文级工程骨架**（双链路、门禁、审计、signoff、批量矩阵），但**尚不具备论文级结论可信性**。阻断原因不在“框架缺失”，而在：

1. 关键门禁存在可绕过路径（失败可被“结构补齐/默认值”掩盖）；
2. paper 路径存在证据语义改写（`failed/absent -> ok, score=0.0`）；
3. 几何主链存在 `sync_primary + relation_digest` 顺序冲突，导致稳定 `mismatch`；
4. 消融矩阵配置与 override 白名单语义不一致，出现全量失败仍可走到 `ALLOW_FREEZE` 的现实风险。

**结论：当前产物不支持论文级主结论直接对外声明。**

## 二、静态代码与现有产物可直接确认的差距项

> 说明：以下每项均已在源码或现有 run 产物中找到直接证据。

### S1（P0）Python 3.10 兼容性阻断
- 结论：成立。
- 证据：`scripts/run_onefile_workflow.py` 第 23 行使用 `from datetime import UTC`（仅 Python 3.11+）。
- 影响：3.10 环境下 onefile 入口直接 ImportError。

### S2（P0）paper 路径将失败证据改写为可评分样本
- 结论：成立。
- 证据：`scripts/run_onefile_workflow.py` 第 672-852 行 `_prepare_detect_record_for_scoring()`；paper 模式下把 `content_evidence_payload.status` 强制改为 `ok`，`score=0.0`。
- 产物证据：`outputs/colab_complete_workflow/colab_run_paper_full_cuda/artifacts/workflow_cfg/detect_record_for_scoring.json` 中确有 `status: "ok"` 与 `score: 0.0`，原始 `detect_record.json` 为 `status: "failed"`。
- 风险：失败语义被“可校准分数”覆盖，削弱拒绝语义。

### S3（P0）attack coverage 可由 onefile 预填充 absent 条目通过
- 结论：成立。
- 证据：
  - `scripts/run_onefile_workflow.py` 第 1539-1600 行 `_ensure_attack_protocol_report_coverage_ready()` 会预填 `metrics_by_attack_condition`，并写入 `status: "absent"`, `onefile_pre_audits_fill: true`；
  - `scripts/audits/audit_attack_protocol_report_coverage.py` 第 228-265 行仅提取 `group_key`，不验证真实执行结果。
- 产物证据：`artifacts/evaluation_report.json` 第 317 行后全部 condition 为 `status: "absent"` 且 `onefile_pre_audits_fill: true`。
- 风险：覆盖审计从“执行覆盖”退化为“键覆盖”。

### S4（P0）experiment_matrix 汇总字段不完整，导致 onefile 阻断条件失效
- 结论：成立。
- 证据：
  - `main/evaluation/experiment_matrix.py` 第 1028-1074 行 `_write_grid_artifacts()` 写出的 `grid_summary.json` 仅含 `executed/results`；
  - `scripts/run_onefile_workflow.py` 第 1092-1125 行 `_load_experiment_matrix_summary()` 以缺省值读取 `total/succeeded/failed=0`；
  - 第 1863-1894 行阻断依赖这些计数。
- 产物证据：`outputs/.../outputs/experiment_matrix/artifacts/grid_summary.json` 顶层只有 `executed`，缺失 `total/succeeded/failed`，但 `results` 实际全 `failed`。
- 风险：应阻断场景可能被误判为“无失败统计”。

### S5（P0）几何链存在 `sync_primary + relation_digest` 顺序冲突（稳定 mismatch）
- 结论：成立（并已在当前产物出现）。
- 证据：
  - `main/watermarking/detect/orchestrator.py` 第 3810-3985 行，`sync_primary` 模式先跑 sync，再门控 anchor；
  - 第 3660-3732 行 `_build_geometry_runtime_inputs()` 仅从 `anchor_result/sync_result` 注入 `relation_digest`；sync-first 时无法提前获得；
  - `main/watermarking/geometry_chain/sync/latent_sync_template.py` 第 912-919 行，v2 强制 `relation_digest`，缺失即 `mismatch`。
- 产物证据：`records/detect_record.json` 出现 `geometry_failure_reason: "relation_digest_missing_for_v2"`、`sync_status: "mismatch"`、`anchor_gated_by_sync_failure`。
- 风险：几何主证据链在该组合下可系统性退化。

### S6（P1）multi-protocol 失败在 paper signoff 下可不阻断
- 结论：成立。
- 证据：
  - `scripts/run_onefile_workflow.py` 第 441 行 `_should_block_on_multi_protocol_validation_error()` 对“failed protocols”返回不阻断；
  - `scripts/run_freeze_signoff.py` 第 798-801 行：`paper` 仅 `--require-compare-summary`，`--require-all-ok` 只在 `publish`。
- 产物证据：
  - `artifacts/multi_protocol_evaluation/.../compare_summary.json` 为 `status: "fail"`；
  - `artifacts/signoff/signoff_report.json` 中 `require_all_ok: false` 且最终 `freeze_signoff_decision: "ALLOW_FREEZE"`。

### S7（P1）onefile 会回填 matrix/repro 锚点默认 digest
- 结论：成立。
- 证据：`scripts/run_onefile_workflow.py` 第 1603-1752 行 `_ensure_experiment_matrix_grid_summary_anchors()` 对缺失 digest 计算 `_fallback_digest(...)` 并回填。
- 风险：缺失证据被“可计算默认值”替代，削弱审计可拒绝性。

### S8（P1）`semantic_saliency_v2` 在默认策略下存在自动 fallback 到 proxy
- 结论：成立。
- 证据：
  - `main/watermarking/content_chain/semantic_mask_provider.py` 第 1015-1065 行 `auto_fallback` 在模型不可用时切到 `proxy_v1`；
  - 第 321-359 行异常分支还可能降级到 `semantic_saliency_v1` / `texture_fallback`；
  - `configs/paper_full_cuda.yaml` 仅设 `mask.impl_id: semantic_saliency_v2`，未强制 `saliency_source=model_v2`。
- 风险：语义显著性主张可能退化为 proxy 语义。

### S9（P1）attention proxy 进入 paper profile 的旧结论需修正
- 结论：**旧结论“paper 下默认走 proxy”不准确；应修正为“paper 下默认禁用 proxy，但 runtime self-attention 缺失会导致几何证据不可用”。**
- 证据：`main/watermarking/detect/orchestrator.py` 第 118-125 行，`paper_faithfulness.enabled=true` 时强制 `enable_attention_proxy=False`。
- 补充：`attention_anchor_extractor` 仍支持 `proxy` 语义（第 600-677 行），但 paper 默认不会走该分支。

### S10（P1）experiment_matrix ablation 轴与 override 白名单语义不一致（当前 run 已全量失败）
- 结论：成立。
- 证据：
  - `configs/paper_full_cuda.yaml` 中 `ablation_variants` 使用 `disable_geometry_chain/disable_content_chain/...`；
  - `main/evaluation/experiment_matrix.py` 第 742 行直接拼接 `ablation.{key}=...`；
  - 实际产物 `grid_summary.json` 的 `failure_reason` 显示 `override_not_allowed: key=ablation.disable_geometry_chain` 等。
- 风险：消融矩阵在当前配置下不可执行，比较结论失真。

### S11（P2）Notebook 配置引导存在阶段性歧义
- 结论：成立（中风险）。
- 证据：`notebook/Colab_Workflow.ipynb` 早期 cell 设 `CONFIG_CHOICE = "default"`，后续第 5 步转为 `paper_full_cuda` 主流程。

### S12（P2）依赖声明分裂，复现环境入口不统一
- 结论：成立（中风险）。
- 证据：`pyproject.toml` 仅含 pytest 配置，不含项目依赖；`requirements.txt` 为大体量环境快照式清单。
- 风险：第三方复现时难以得到“最小且确定”的运行依赖集合。

## 三、需要真实 SD3.5 workflow 才能确认的差距项

> 以下结论不能仅靠静态代码得出，必须在真实模型与真实攻击协议下验证。

1. 是否显著超越 T2SMark（同样本、同攻击预算、同阈值策略）。此项风险需要真实 SD 模型 workflow 才能判断。  
2. 画质—鲁棒性权衡是否成立（LPIPS/PSNR/SSIM/FID 与检测指标联立）。此项风险需要真实 SD 模型 workflow 才能判断。  
3. 几何攻击强度递增下失败语义是否自然（rotate/resize/crop/translate，FNR 单调性）。此项风险需要真实 SD 模型 workflow 才能判断。  
4. NP 校准稳定性是否充分（跨 seed/prompt/attack 强度；当前产物 `calibration_samples=1`，统计不足）。此项风险需要真实 SD 模型 workflow 才能判断。  
5. runtime self-attention 捕获稳定性与收益是否可重复。此项风险需要真实 SD 模型 workflow 才能判断。

## 四、风险优先级（最终）

### P0（立即修复，否则论文主流程可信性不足）
1. `from datetime import UTC` 的 Python 3.10 兼容问题。  
2. paper 路径禁止 `failed/absent -> ok, score=0.0` 的 scoring fallback。  
3. attack coverage 审计改为“执行状态 + 证据”校验，禁止 onefile 预填充替代执行。  
4. 修复 experiment_matrix `grid_summary` 计数字段写出，确保 onefile 的零成功阻断生效。  
5. 修复 `sync_primary + relation_digest` 顺序冲突（需在 sync 前提供独立 relation_digest 或调整执行序）。

### P1（论文实验前完成）
1. 修复 ablation 轴键名与 whitelist 语义不一致（当前导致 matrix 全失败）。  
2. paper signoff 将 multi-protocol 主协议失败设为阻断（至少 `paper` 启用 `require_all_ok` 或等价 gate）。  
3. 取消 matrix/repro 缺失锚点的默认 digest 回填，改为硬失败。  
4. 固化 semantic saliency 源策略（paper 禁止自动 proxy fallback）。  
5. 扩展消融轴（LF-only/HF-only、mask source、sync 参数扫描、attention 门限、连续强度谱）。

### P2（投稿前增强）
1. Notebook 配置路径去歧义（避免 `default` 与 `paper_full_cuda` 双语义）。  
2. 依赖入口标准化（最小运行依赖与开发依赖分离）。

## 五、对“是否有利于后期消融实验”的最终判断

当前**不利于论文级消融**。主要原因：
- matrix 轴定义与 override 规则失配导致大规模失败；
- coverage 与锚点存在“可补齐”路径，破坏“失败可拒绝性”；
- 全失败统计在某些路径可能未被有效阻断，造成“流程可完成 ≠ 结果可比较”。
