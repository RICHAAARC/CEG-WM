# 项目风险与研究差距综合审计报告

> 审计版本号：2026-03-01-1850  
> 审计方式：静态源码审计 + 现有运行产物复核（`outputs/colab_complete_workflow/colab_run_paper_full_cuda`）  
> 审计目标：按“静态可确认差距项 / 真实 SD workflow 才能判定项 / 后期消融实验判断项”给出可执行整改结论。

---

## 静态可确认差距项

### 1) HIGH-CRITICAL：Python 3.10 兼容性阻断（onefile 入口）

**差距项说明**
- onefile 入口使用 `from datetime import UTC`，该符号依赖 Python 3.11+。

**证据位置**
- 代码：`scripts/run_onefile_workflow.py`（`from datetime import UTC`）

**影响说明**
- 在 Python 3.10 环境中，onefile 入口可直接 ImportError，流程无法启动。

---

### 2) HIGH-CRITICAL：paper 路径存在失败证据改写为可评分样本的分支

**差距项说明**
- `_prepare_detect_record_for_scoring()` 可将 `content_evidence_payload.status` 改写为 `ok` 并回填分数（含 `score=0.0` 场景）。

**证据位置**
- 代码：`scripts/run_onefile_workflow.py`（`_prepare_detect_record_for_scoring()` 相关分支）
- 产物：
  - `outputs/colab_complete_workflow/colab_run_paper_full_cuda/artifacts/workflow_cfg/detect_record_for_scoring.json`
  - 对照 `records/detect_record.json`

**影响说明**
- 失败语义可能被“可校准分数”覆盖，削弱拒绝语义与校准样本纯度。

---

### 3) HIGH-CRITICAL：attack coverage 可被 onefile 预填 absent 条目通过

**差距项说明**
- onefile 可预填 `metrics_by_attack_condition`（`status="absent"` + `onefile_pre_audits_fill=true`），而覆盖审计仅检查 `group_key` 键集合。

**证据位置**
- 代码：
  - `scripts/run_onefile_workflow.py`（`_ensure_attack_protocol_report_coverage_ready()`）
  - `scripts/audits/audit_attack_protocol_report_coverage.py`（仅提取 `group_key`）
- 产物：`artifacts/evaluation_report.json`（大量 `status: "absent"` 且 `onefile_pre_audits_fill: true`）

**影响说明**
- 覆盖审计可能从“执行覆盖”退化为“键覆盖”，提升伪通过风险。

---

### 4) HIGH-CRITICAL：experiment_matrix 汇总字段缺失可导致 onefile 阻断条件失效

**差距项说明**
- `grid_summary.json` 未统一输出 `total/succeeded/failed` 时，onefile 读取缺省值并可能误判阻断条件。

**证据位置**
- 代码：
  - `main/evaluation/experiment_matrix.py`（`_write_grid_artifacts()` 输出字段）
  - `scripts/run_onefile_workflow.py`（`_load_experiment_matrix_summary()` 读取与阻断逻辑）
- 产物：`outputs/.../outputs/experiment_matrix/artifacts/grid_summary.json`

**影响说明**
- 应阻断的失败矩阵可能被判定为“无失败统计”，破坏流程门禁。

---

### 5) HIGH-CRITICAL：几何链 `sync_primary + relation_digest` 顺序冲突导致稳定 mismatch

**差距项说明**
- `sync_primary` 先 sync 后 anchor 的路径下，`relation_digest` 可能尚未建立；v2 同步模块要求 `relation_digest`，缺失即 mismatch。

**证据位置**
- 代码：
  - `main/watermarking/detect/orchestrator.py`（`sync_primary` 路径与 runtime inputs 构造）
  - `main/watermarking/geometry_chain/sync/latent_sync_template.py`（v2 对 `relation_digest` 要求）
- 产物：`records/detect_record.json`（`relation_digest_missing_for_v2`、`sync_status: mismatch`）

**影响说明**
- 几何主证据链可系统性退化，双证据融合稳定性下降。

---

### 6) HIGH：multi-protocol 失败在 paper signoff 下可不阻断

**差距项说明**
- paper signoff 仅要求 compare summary，不强制 all-ok；multi-protocol 失败可在某些条件下继续。

**证据位置**
- 代码：
  - `scripts/run_onefile_workflow.py`（`_should_block_on_multi_protocol_validation_error()`）
  - `scripts/run_freeze_signoff.py`（paper 与 publish 约束差异）
- 产物：
  - `artifacts/multi_protocol_evaluation/.../compare_summary.json`（`status: fail`）
  - `artifacts/signoff/signoff_report.json`（`require_all_ok: false`，`ALLOW_FREEZE`）

**影响说明**
- 失败协议可能进入冻结放行路径，降低发布门禁刚性。

---

### 7) HIGH：onefile 会回填 matrix/repro 锚点默认 digest

**差距项说明**
- 缺失锚点时通过 `_fallback_digest(...)` 生成默认值并回填。

**证据位置**
- 代码：`scripts/run_onefile_workflow.py`（`_ensure_experiment_matrix_grid_summary_anchors()`）

**影响说明**
- 缺失证据可能被“可计算默认值”掩盖，降低审计可拒绝性。

---

### 8) HIGH：`semantic_saliency_v2` 在默认策略下可自动降级到 proxy

**差距项说明**
- `auto_fallback` 与异常分支可将语义来源降级到 proxy/texture 路径。
- 配置若未强制 `saliency_source=model_v2`，则主张语义存在漂移风险。

**证据位置**
- 代码：`main/watermarking/content_chain/semantic_mask_provider.py`（`auto_fallback` 与异常降级分支）
- 配置：`configs/paper_full_cuda.yaml`（仅 `mask.impl_id`，未强制 source 的情形）

**影响说明**
- 论文所述“语义显著性来源”可能被代理路径替代，方法真实性受损。

---

### 9) HIGH：paper 默认禁用 attention proxy，若 runtime self-attention 缺失几何证据可不可用

**差距项说明**
- paper 模式下强制 `enable_attention_proxy=false`，当 runtime 自注意力不可得时，几何辅证据路径缺少降级空间。

**证据位置**
- 代码：`main/watermarking/detect/orchestrator.py`（paper 模式关闭 proxy）
- 代码：attention extractor 仍存在 proxy 语义分支，但 paper 默认不走

**影响说明**
- 在 runtime self-attention 缺失场景下，几何证据可用性风险上升。

---

### 10) HIGH：ablation 轴定义与 override 白名单语义不一致（现有 run 已全量失败）

**差距项说明**
- `ablation_variants` 使用的 key 与 override 白名单/规范化路径不一致，导致矩阵任务被拒绝。

**证据位置**
- 配置：`configs/paper_full_cuda.yaml`（`ablation_variants` 字段）
- 代码：`main/evaluation/experiment_matrix.py`（`ablation.{key}` 拼接）
- 产物：`grid_summary.json`（`override_not_allowed: key=ablation.disable_*`）

**影响说明**
- 消融矩阵不可执行，结果不可比较。

---

### 11) MEDIUM：Notebook 配置引导存在阶段性歧义

**差距项说明**
- 早期使用 `CONFIG_CHOICE="default"`，后续切换到 `paper_full_cuda`，流程理解成本与误操作风险上升。

**证据位置**
- `notebook/Colab_Workflow.ipynb`

**影响说明**

---

### 12) MEDIUM：依赖声明分裂，复现入口不统一

**差距项说明**
- `pyproject.toml` 未承载项目运行依赖，`requirements.txt` 为快照式清单，缺少最小可复现依赖面。

**证据位置**
- `pyproject.toml`
- `requirements.txt`

**影响说明**
- 第三方复现环境稳定性与可移植性下降。

## 真实 SD workflow 才能判定项

### 1) 是否显著超越 T2SMark（同样本、同攻击预算、同阈值策略）

**需要 workflow 的原因**
- 该结论依赖真实样本分布与严格对照实验，静态代码与单次产物无法证明统计优势。

### 2) 画质—鲁棒性权衡是否成立

**需要 workflow 的原因**
- 需联合观测 LPIPS/PSNR/SSIM/FID 与检测指标，属于运行期统计行为，无法通过静态阅读得出。

### 3) 几何攻击强度递增下失败语义是否自然且单调

**需要 workflow 的原因**
- rotate/resize/crop/translate 的真实退化曲线与 FNR 单调性必须通过攻击 sweep 验证。

### 4) NP 校准稳定性是否充分

**需要 workflow 的原因**
- 跨 seed/prompt/attack 强度的阈值稳定性需要多样本实测；当前 `calibration_samples=1` 仅能说明不足，不能说明最终可行性。

### 5) runtime self-attention 捕获稳定性与收益是否可重复

**需要 workflow 的原因**
- hook 触发率、张量质量、对几何链收益均为运行期性质，需多轮真实推理复验。

## 后期消融实验判断项

### 1) 有利因素（结构层）

- 已定义 experiment matrix 与多条消融轴，具备开展消融的工程框架基础。

### 2) 当前主要阻断

- 轴定义与白名单语义失配，导致矩阵任务大规模失败。
- coverage 与锚点存在“可补齐”路径，削弱失败可拒绝性。
- 失败语义可能被分数化改写，影响消融差异解释。
- 某些失败路径在 signoff 规则下未被强阻断。

### 3) 结论

- 当前状态不利于论文级消融，尚不满足“可运行 + 可比较 + 可审计拒绝”的三重条件。

## 建议修复顺序

### P0（先修阻断与真实性）

1. 修复 Python 3.10 兼容性入口问题（onefile 可启动）。
2. 关闭或收紧失败证据分数化改写路径（防止 failed→ok 语义漂移）。
3. 修复 `sync_primary + relation_digest` 顺序冲突，确保几何链可稳定运行。
4. 修复 experiment_matrix 汇总字段契约，恢复 onefile 阻断有效性。

**目标验收**
- onefile 在目标环境可启动。
- 失败语义不再被隐式改写为可评分样本。
- 几何链不再稳定出现 `relation_digest_missing_for_v2` 系统性 mismatch。
- 阻断逻辑对失败矩阵生效。

### P1（再修门禁与审计刚性）

5. 强化 attack coverage 审计：从“键覆盖”升级为“执行覆盖 + 状态约束”。
6. 收紧 paper signoff 对 multi-protocol 失败的放行规则。
7. 禁止或显式标识锚点 fallback digest 回填用于正式审计路径。
8. 固化 `semantic_saliency_v2` 来源策略（避免 proxy 漂移）。

**目标验收**
- coverage 审计无法由 absent 预填充通过。
- paper signoff 对关键失败具备阻断能力。
- 审计锚点缺失可拒绝，不能以默认值伪通过。

### P2（统计与复现质量）

9. 修复 ablation 轴与 override 白名单语义一致性，恢复消融矩阵可执行性。
10. 统一 notebook 配置引导路径与依赖入口，降低复现歧义。
11. 扩充样本并完成真实 SD workflow 统计验证，形成可发布实证结论。

**目标验收**
- 消融矩阵稳定执行且失败原因可解释。
- 复现说明与环境依赖入口统一。
- 关键指标在真实 workflow 上可复现并满足统计门槛。
