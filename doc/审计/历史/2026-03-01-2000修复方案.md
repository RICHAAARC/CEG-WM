## 3. 修复项明细

### FIX-P0-1：替换 `np.corrcoef` 为安全实现

**问题**
- `np.corrcoef(channels)` 在 Windows/NumPy 组合下出现 Fatal Python error: Aborted。

**操作文件**
- `main/watermarking/detect/orchestrator.py`

**具体修复操作**
1) 在 `_build_attention_maps_from_latents` 前新增 `_safe_corrcoef()`：
```python
def _safe_corrcoef(channels: np.ndarray) -> np.ndarray:
    mat = channels.astype(np.float64)
    mat = mat - mat.mean(axis=1, keepdims=True)
    norms = np.sqrt((mat * mat).sum(axis=1, keepdims=True))
    norms = np.where(norms < 1e-10, 1.0, norms)
    mat = mat / norms
    return mat @ mat.T
```
2) 替换调用：
```python
# 前
correlation = np.asarray(np.corrcoef(channels), dtype=np.float64)
# 后
correlation = np.asarray(_safe_corrcoef(channels), dtype=np.float64)
```

**验证**
- `pytest tests/test_anchor_gate_by_sync_status.py -v` 不再 Aborted。

**审计留痕**
- 不改 `relation_digest` 输入语义，不触及冻结合约字段。

### FIX-P0-2：保障 DCT 块切片连续性

**问题**
- HWC 切片在 channel 维上得到非连续视图，传入 `_dct2` 可能触发底层 BLAS Abort。

**操作文件**
- `main/watermarking/content_chain/low_freq_coder.py`

**具体修复操作**
```python
# 前
block = work[r0:r0 + block_size, c0:c0 + block_size, channel_idx]
coeff = _dct2(block, dct_matrix)

# 后
block = np.ascontiguousarray(
    work[r0:r0 + block_size, c0:c0 + block_size, channel_idx]
)
coeff = _dct2(block, dct_matrix)
```

**验证**
- `pytest tests/test_c3_real_embedding_pipeline.py -v` 不再 Aborted。

**审计留痕**
- 仅内存布局修复，不改变数值定义。

