#### 关键阻断点

**当前阻断（主链不自足）：** `paper_full_cuda.yaml` 无 `embed.input_image_path`。`main/` 独立运行时，`_resolve_embed_input_image_path()` 三个候选均缺失，返回 `None`；`_build_content_inputs_for_embed()` 返回 `None`；`SemanticMaskProvider` 以空 inputs 调用，返回 `status="failed"`, `content_failure_reason="mask_extraction_no_input"`。当前产物可达性来自 `scripts/` 注入。

**语义有效性存疑（即使输入路径修复）：** 当前 embed 输入为 512×512 线性颜色梯度合成图。InSPyReNet 对合成梯度图提取的"显著性掩码"不等价于真实语义场景中的显著性，"语义自适应（Semantic Saliency-Adaptive）"声明仅在输入为真实语义图像时才成立。**【方法更新 2026-03-08】** 正确修复策略为 **Preview Generation（预览生成）**：embed 前以相同条件 $(p, s)$ 先执行一次无注入 SD3 推理得到 $x_{\text{preview}} = G_\theta(p, s)$，以此作为 `SemanticMaskProvider` 语义掩码输入；由于预览生成与正式生成条件完全相同，语义结构先验一致，"语义自适应"声明在此框架下成立，且无需维护仓库内预存图像文件（见 `doc/方法.md` 第（三）节）。

**若当前阻断修复后，下一个阻断点：**
1. 语义模型权重在目标运行环境是否真实可用（属 § 五 runtime 判定项）；
2. planner 在全部目标样本上是否稳定产出非空 `plan_digest`；
3. 真实语义图像上的掩码是否使 LF/HF 注入信号可被有效检出（与链路 B 阻断相关）。

#### 链路 A 结论

> **产物字段表面可达**（`mask_digest`、`plan_digest` 非 null，`injection_mode = "subspace_projection"`），**但主链不自足**；可达性依赖 `scripts/` 注入的合成梯度图，语义有效性存疑。**修复方向已明确为 Preview Generation**（$x_{\text{preview}} = G_\theta(p, s)$）：在 `scripts/run_onefile_workflow.py:_ensure_default_embed_input_image()` 中改为执行无注入 SD3 推理取代合成梯度生成，同时解决"输入路径不自足"与"语义有效性存疑"两个问题；见 `doc/方法.md` 第（三）节。`configs/ + main/` 独立运行路径仍有待修复。

#### 关键问题

**Q1：LF BP 未收敛**（`lf_status_degraded_reason = "bp_not_converged"`）

LF 检测依赖 sparse_ldpc 码字通过 BP 解码恢复，50 次迭代后仍未收敛。比特准确率 0.484375 低于随机基准（0.5）。LF 水印信号实质上无效。可能根因：embed 侧 LF 注入基于合成梯度图的掩码频率分配，与真实图像频率特性不匹配，SD3 去噪过程对 LF 频带的改变超出了码字容量。

**Q2：HF 评分机制对 SD3 图像无判别性**（**直接导致链路 D FPR=1.0**）

⚠ **【复核修正（2026-03-08）】：** 原表述"`hf_score_raw(-0.0006)` → 规范化公式放大 → `hf_score(262.4)`，约 45 万倍"为事实性错误，两者来自独立计算路径（见上方 HF 子通道复核注）。`content_score` 中 99.8% 来自 HF 贡献。正确根因：UnifiedContentExtractor 的 `compute_hf_score()` 基于 SD3 推理轨迹 L2 范数统计（`observed = mean_l2+std_l2 ≈ 262`），`embed_hint=None`（embed_record.hf_score=None）→ `boost=1.0`，该值对 watermarked 与 non-watermarked 的 SD3 图像均近似相等，导致 hf_score 分布高度重叠、FPR=1.0。修复方向（`compute_hf_score` 轨迹 L2 范数设计）成立，但问题根因是轨迹 L2 范数对两类图像缺乏判别性，而非"放大"机制本身。

**若当前阻断修复后，下一个阻断点：**
1. LF 收敛性需使用真实语义图像重新验证（属 § 五 runtime 判定项）；
2. HF 分数规范化公式需重新设计，使其对 watermarked 与 non-watermarked 图像具有统计可分离性；
3. 即使 LF/HF 双 ok，最终判决阈值来源口径仍需统一（见链路 C）。
