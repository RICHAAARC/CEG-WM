# 2026-03-06-1530 修复方案

> 对应核验：`doc/审计/核验结果报告.md`
> 事实基准：`doc/审计/2026-03-06-1530审计报告.md`
> 原则：最小改动、低风险、保持 workflow 产物完整、保持冻结语义与门禁约束

## 1. 整体思路

1. 先修 `P0` 基础可达性与样本纪律，再修 `P1` 字段一致性。
2. 不改核心判决算法，不改冻结字段语义，不新增写盘旁路。
3. 全程保持 `embed/detect/calibrate/evaluate` 四阶段可产物、可审计。

## 2. 详细实施方案（按优先级）

### 2.1 P0：修复 `GAP-D1`（dual_branch 忽略 `pair_count`）

文件：`scripts/run_onefile_workflow.py`

修改点：

1. 函数 `_prepare_detect_records_with_minimal_ground_truth()` 的 dual_branch 分支按 `pair_count` 生成正负样本对，不再固定输出 1 对。
2. 保持命名兼容：
   - `pair_count == 1`：继续写 `detect_records_{stage}_gt_positive.json` / `detect_records_{stage}_gt_negative.json`
   - `pair_count > 1`：按现有 clone 分支模式写入带索引文件
3. 保留现有字段语义：`label`、`ground_truth`、`is_watermarked`、`ground_truth_source`、`calibration_sample_usage`。
4. 明确 dual_branch 的部分失败语义：
   - 当某个 pair 的 dual_branch 调用失败时，不静默回退为普通样本；该 pair 的 negative 样本必须写入 `calibration_sample_usage="synthetic_negative_for_ground_truth_closure"`，并追加 `dual_branch_failure_reason` 审计字段。
   - 当全部 pair 均 dual_branch 失败时，允许全量退回 clone fallback，并输出 WARN 日志，保留当前可运行性。

预期效果：

- `calibration_samples` 不再固定为 1。
- `evaluate.n_pos/n_neg` 可按配置规模增长。
- `threshold_value` 来源由单样本退化转为真实 null 分布。

### 2.2 P0：修复 `GAP-B1`（LF PRC latent 维度不足）

文件：

- `main/watermarking/content_chain/subspace/subspace_planner_impl.py`
- `main/watermarking/content_chain/low_freq_coder.py`

修改点：

0. 修复口径以真实实现为准：`block_length` 由 `main/watermarking/content_chain/ldpc_codec.py` 的 `max(message_length + 8, round(message_length * 1.5))` 决定；不采用 `message_length × (1 + ecc_sparsity)` 的旧推导口径。
1. `LFCoderPRC._flatten_recursive_to_list()`：补充对 `numpy.ndarray` 与 `torch.Tensor` 的扁平化支持，确保 detect 侧 `final_latents` 可被正确展平并参与 `block_length` 检查。
2. `LFCoderPRC.detect_score()`：在返回 trace 中追加 append-only 审计字段：
   - `available_latent_dim`
   - `required_block_length`
   并保持既有 `status` / `lf_failure_reason` 语义不变。
3. `SubspacePlannerImpl._build_executable_basis_payload()`：保留“仅 LF 通道取消 `max_cols=16`”作为**次级硬化项**。
4. 不在 planner 层新增 `basis_dim < expected_block_length` 强 fail-fast：当前 `effective_rank` 受 `rank` 控制，直接 gate 会把可运行路径转为常态失败，不符合最小改动原则。
5. 不修改 `status` 与 `lf_failure_reason` 枚举定义。

预期效果：

- detect 侧真实 tensor latents 可进入 PRC 解码分支，不再因类型兼容问题误报 `lf_insufficient_latent_dimension`。
- 当维度确实不足时继续 `failed`，且可通过 `available_latent_dim` / `required_block_length` 直接审计定位。

### 2.3 P1：修复 `GAP-B2`（`lf_status` 与 `lf_score` 冲突）

文件：`main/watermarking/detect/orchestrator.py`

修改点：

1. 函数 `_bind_raw_scores_to_content_payload()` 中，取消 `score_parts["lf_status"]` 的无条件 PRC trace 覆写。
2. 保留 `score_parts["lf_detect_trace"]`。
3. 追加字段 `score_parts["prc_latent_status"]`（append-only），仅表达 PRC 原始路径状态。
4. `score_parts["lf_status"]` 保持 ContentDetector 主语义，不改变字段名称与语义。

预期效果：

- `lf_score` 与 `lf_status` 语义一致。
- PRC 路径状态仍可审计追踪。

### 2.4 P1：对齐 detect 端 `mask_digest` 验收口径

文件：`main/watermarking/detect/orchestrator.py`

修改点：

1. detect 成功路径下，若 `content_evidence_payload.mask_digest` 为空，且 `input_record.content_evidence.mask_digest` 非空，则透传填充。
2. 不新增替代语义，不改变 `status` 语义。

预期效果：

- 对齐验收条件：detect 端 `content_evidence_payload.status == "ok"` 且 `mask_digest != null`。

### 2.5 P1：收紧 synthetic negative 过滤配置（封闭 FAIL-4 风险）

文件：`configs/paper_full_cuda.yaml`

补充实现文件：`main/watermarking/detect/orchestrator.py`

修改点：

1. 在 `calibration:` 下设置 `exclude_synthetic_negative_closure_marker: true`。
2. 在 `evaluate:` 下设置 `exclude_synthetic_negative_closure_marker: true`。
3. 在 `_load_records_for_evaluate()` 增加 opt-in pre-filter，读取 `evaluate.exclude_synthetic_negative_closure_marker` 并过滤 `content_evidence_payload.calibration_sample_usage == "synthetic_negative_for_ground_truth_closure"`（当前 evaluate 路径尚未读取该配置，需补齐代码支撑）。
4. 不新增配置键，仅将既有开关显式设为强制过滤；默认值保持 `False` 以维持向后兼容。

预期效果：

- 即使 dual_branch 的部分 pair 回落到 synthetic negative，相关样本也会被排除出正式校准/评估统计。
- `sampling_policy.n_rejected_synthetic_negative_closure` 可用于审计追溯。

## 3. 注意事项与约束关系

### 3.1 冻结面与门禁面约束

1. 不改变冻结字段语义、枚举含义、默认值解释（尤其 `status`、`decision_status`、`absent/mismatch/failed/error`）。
2. 新增字段仅 append-only：本方案仅新增 `score_parts.prc_latent_status`。
3. 不新增写盘旁路，不绕过：
   - `path_policy`
   - `runtime_whitelist`
   - `freeze_gate`
4. evaluate 阶段阈值只读，不重估阈值。
5. 预期变更声明：若执行 LF basis payload 硬化（例如 LF 通道取消固定截断），`plan_digest` 可能发生变化；该变化属于实现修复导致的可解释变更，不视为语义漂移。

### 3.2 workflow 产物完整性约束

1. 不允许通过删除产物、跳过阶段、静默吞错达成“跑通”。
2. 保持四阶段最小文件集合持续可生成：
   - `records/embed_record.json`
   - `records/detect_record.json`
   - `records/calibration_record.json`
   - `records/evaluate_record.json`

## 4. 执行计划

1. 第 1 步：修改 `scripts/run_onefile_workflow.py`，修复 dual_branch 的 `pair_count` 批量生成。
2. 第 2 步：在 `run_onefile_workflow.py` 补充 dual_branch 部分失败语义（pair 级 synthetic marker + failure reason）。
3. 第 3 步：优先修复 LF detect latent 类型兼容（`low_freq_coder.py` 扁平化支持 `ndarray/tensor`）并追加 `available_latent_dim` / `required_block_length` 审计字段。
4. 第 4 步：将 LF 通道 `max_cols=16` 调整为仅 HF 保留或 LF 取消截断（硬化项，不加强制 fail-fast，不调整 message_length/feature_dim）。

6. 第 6 步：补齐 detect 端 `mask_digest` 透传。
7. 第 7 步：在 `orchestrator.py` 的 `_load_records_for_evaluate()` 落实 synthetic negative pre-filter，并同步 `paper_full_cuda.yaml` 开启 evaluate 开关。
8. 第 8 步：执行定向测试与全量回归，验证门禁与产物完整性。

## 5. 验收标准与回归测试

### 5.1 验收标准

1. `injection_rule_summary.injection_mode == "subspace_projection"`。
2. `embed.mask_digest != null` 且 `embed.plan_digest != null`。
3. detect 端 `content_evidence_payload.status == "ok"` 且 `mask_digest != null`。
4. 校准阈值 `threshold_value` 非退化占位值（例如 `-1.0`）。
5. `evaluate.n_pos > 0` 且 `evaluate.n_neg > 0`，并可计算 `tpr_at_fpr_primary`。
6. 最终统计口径要求 `fpr_empirical <= target_fpr`（以修复后有效样本集为准）。
7. `calibration` 与 `evaluate` 的 `exclude_synthetic_negative_closure_marker` 必须为 `true`。
8. dual_branch 部分失败场景下，失败 pair 必须包含 `dual_branch_failure_reason` 且对应样本不进入正式统计。
