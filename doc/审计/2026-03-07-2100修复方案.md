### 2.2 NEW_BLOCKER_3 修正（事实澄清）

**原始声明：** "detect 侧缺乏 `__detect_final_latents__` 注入机制"

**澄清结论：** 该声明不成立。

**代码证据（`main/cli/run_detect.py`，第 555–637 行）：**

当 `pipeline_obj is not None` 时：

```
else:  # pipeline 存在
    inference_result = infer_runtime.run_sd3_inference(
        cfg, pipeline_obj, device, seed,
        ...,
        capture_final_latents=True,   ← 显式捕获 final_latents
        ...
    )
    final_latents = inference_result.get("final_latents")

if final_latents is not None:
    cfg["__detect_final_latents__"] = final_latents   ← 注入机制完整
```

**产物证据：** detect_record 中 `geometry_evidence_payload.status = "ok"` 且 `geo_score = 0.931`（几何链 latent sync 需消费 `__detect_final_latents__`），证明 latents 确实已在 Colab GPU run 中被捕获并注入。

**真实阻断（更正）：** `detect_runtime_mode = "fallback_identity_v0"` 的根因并非 latents 缺失，而是：

1. Chain A bug → detect pre-plan 获得 `mask_digest = null` → subspace_planner 使用 fallback plan → `lf_basis` 无效
2. `detector_scoring.extract_lf_score_from_detect_latents(final_latents, lf_basis=<fallback_basis>, ...)` → `detect_lf_status != "ok"`
3. `detect_runtime_mode` 赋值门控中 `detect_lf_status == "ok"` 条件不满足

**修复路径：** 修复 Chain A（`parents[3]`）→ detect 侧有效 mask_digest → 真实 subspace plan → 正确 `lf_basis` → `detect_lf_status = "ok"`（需实测验证）

## 三、修复方案整体思路

### 3.1 根本阻断因果链

```
semantic_mask_provider.py:877
  parents[2] = main/  ← 单点 bug
    ↓ probe_failure_reason: path_not_found
    ↓ mask_digest = null
    ↓ SubspacePlanner fallback plan（无有效 lf_basis）
      ↓ [embed] injection_mode = "latent_direct_fallback"
      ↓ [detect pre-plan] lf_basis 无效
        ↓ detect_lf_status != "ok"
        ↓ detect_runtime_mode = "fallback_identity_v0"
        ↓ content_evidence_payload.status = "failed"
          ↓ fusion decision_status = "error"
          ↓ TPR/FPR 基于 HF 旁路分数（语义不符 + 样本量不足）
```

除上述因果链外，还存在以下独立阻断：

- **embed 输入路径依赖 `scripts/` 注入**（不依赖 P0 的独立缺口）
- **detect CLI pre-plan 无 image 输入**（P0 修复后仍存在，影响 detect 侧 injection context 语义对齐）
- **统计样本量 30 < 100**（P0 修复后仍需扩充）

### 3.2 最小风险修复路径

按优先级排序：

| 优先级 | 修复标签 | 可改变内容 | 触发下一阻断 |
|--------|----------|-----------|-------------|
| P0 | `MASK_PATH_BUG` | 工程根路径解析 | 模型权重实际可加载性 |
| P1a | `EMBED_INPUT_PATH` | embed 独立可达性 | 无 |
| P1b | `DETECT_PREPLAN_INPUT` | detect-side injection 语义对齐 | detect_lf_status 实测 |

| P2 | `STAT_SAMPLE_COUNT` | FPR 粒度 | Colab 运行时长 |
| P3 | `GT_BUNDLE_MAIN` | scripts 依赖解耦 | 集成测试完整性 |

## 七、统计口径一致性分析

### 7.1 Neyman–Pearson calibration 协议保持

修复不改变以下内容：

| 统计属性 | 当前状态 | 修复后状态 |
|----------|---------|-----------|
| FPR 控制方（Neyman–Pearson） | `target_fpr = 0.01` | 不变 |
| 阈值来源 | `main/evaluation/calibrate.py` 单次 calibrate 产出 | 不变 |
| evaluate 阶段禁止重估阈值 | 已冻结 | 不变 |
| `content_score` 字段路径 | `content_evidence_payload.score_parts.*` | 不变（路径不变，内容从降级分数变为目标分数） |

### 7.2 分数语义变更声明

P0 修复后，`content_score` 的底层分数将从 **image domain HF 旁路分数**（当前）切换至 **subspace LF+HF 联合分数**（修复后目标）。

这是 **语义修正**（使分数符合论文声明），而非统计口径变更。

**关键约束：**
