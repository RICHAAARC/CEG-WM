- `models/inspyrenet/ckpt_base.pth` **存在**，大小 367,520,613 字节（~367 MB）
- 失败根因是**模型加载逻辑错误**，与文件是否存在无关

**代码级根因（精确到行号）**：

1. **`_probe_model_v2_availability` 假阳性**（`semantic_mask_provider.py:851-878`）：
   - 仅执行 `torch.load(model_file, map_location="cpu")` → `ckpt_base.pth` 是标准 state_dict，`torch.load()` 返回 dict，不抛异常 → 误判为 True
   - 代码进入实际推理路径而非"不可用"短路

2. **`_load_saliency_model` 将 state_dict 直接返回为"模型对象"**（`semantic_mask_provider.py:937-942`）：
   ```python
   loaded_obj = torch.load(str(model_file), map_location="cpu")
   model_obj = loaded_obj
   if isinstance(loaded_obj, dict) and "model" in loaded_obj:
       model_obj = loaded_obj.get("model")  # ckpt_base.pth 顶层无 "model" key
   # → model_obj = state_dict dict（非 nn.Module）
   ```

3. **`_run_saliency_inference` 在 dict 上调用 `callable()` 失败**（`semantic_mask_provider.py:970`）：
   - `callable(dict)` = `False` → `raise RuntimeError("saliency model is not callable")` → 外层 catch → `fallback_reason = "semantic_model_unavailable: RuntimeError"`

4. **配置缺失 `semantic_model_source`**（`paper_full_cuda.yaml`）：
   - `semantic_mask_provider.py:771`：`mask_cfg.get("semantic_model_source", SEMANTIC_MODEL_BACKEND_BASNET)` → 默认 `"basnet"`
   - `_normalize_semantic_model_source("model_v2")` 映射为 `SEMANTIC_MODEL_BACKEND_BASNET`（`semantic_mask_provider.py:831`）
   - backend 为 `"basnet"` 而非 `"inspyrenet"`，影响推理输出格式解析

**完整调用链**：
```
paper_full_cuda.yaml: saliency_source="model_v2", semantic_model_path="../models/inspyrenet/ckpt_base.pth"
  → _probe_model_v2_availability(): torch.load() 成功（state_dict） → True（假阳性）
  → build_semantic_saliency_mask_v2() → _load_saliency_model()
  → torch.load() = state_dict dict → "model" key 不存在 → model_obj = dict
  → _run_saliency_inference(model=dict): callable(dict) = False
  → RuntimeError("saliency model is not callable")
  → saliency_source==MODEL_V2 → 禁止降级 → status="failed"
                                              content_failure_reason="saliency_source_model_v2_runtime_failed"
```

**需修复的两个独立问题（当前记录在案，暂不修复）**：
1. `_load_saliency_model`：当 backend="inspyrenet" 时，需实例化 InSPyReNet 网络架构，再 `model.load_state_dict(state_dict)` 加载权重
2. `paper_full_cuda.yaml` 的 `mask` 节下补充 `semantic_model_source: "inspyrenet"` 字段

**修复后的下一个阻断点**：
- model 加载成功后，InSPyReNet 推理在 CUDA 环境可能存在版本兼容性问题（→ SD-RUNTIME-1）
