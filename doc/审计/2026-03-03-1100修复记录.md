### B.4 【P0-2/3(1600)】Colab 端缺失输入回溯修复

**问题编号**：mask_extraction_no_input（Colab paper_full_cuda 工作流）

**问题现象**：
```json
{
  "content_evidence_payload": {
    "status": "failed",
    "content_failure_reason": "mask_extraction_no_input"
  }
}
```

detect 阶段虽返回码为 0，但 content_extractor 接收的 inputs=None，导致后续 calibrate/evaluate 阶段无法继续。

**根本原因链**：
1. Embed 阶段成功加载图片并产出 embedding，但**未保存输入图像路径信息**
2. Detect 阶段尝试多个输入来源：
   - cfg["__detect_final_latents__"] → 未存在（detect inference 未产出或 pipeline=None）
   - input_record.image / input_record.latent → 未存在（embed 未保存）
   - image_path 解析 → **在 Colab 环境失败**（绝对路径已过期）
   - 无 fallback → inputs=None
3. 结果：SemanticMaskProvider.extract(cfg, inputs=None) 触发第一道门控 → mask_extraction_no_input

**修復操作**（三部分）：

#### 修復 1：Embed 侧保存输入路径
- 文件：`main/cli/run_embed.py`
- 行号：505-520
- 变更点：在 embed_record 中增加 `inputs` 字段，保存 `input_image_path`

```python
inputs_record = {}
if isinstance(input_image_path, str) and input_image_path.strip():
    inputs_record["input_image_path"] = input_image_path.strip()
if inputs_record:
    record["inputs"] = inputs_record
```

- 设计原理：Embed 阶段已成功加载图片，知道真实有效路径；将此路径保存供下游使用
- 完全 append-only，不改动既有字段

#### 修復 2：Detect 侧增强输入回溯
- 文件：`main/watermarking/detect/orchestrator.py`
- 行号：1230-1310
- 变更点：在 `_build_content_inputs_for_detect()` 中补充 fallback 链，从 embed_record 读取保存的 input_image_path

```python
# 新增 fallback 链（仅在标准来源都失败时启用）：
if isinstance(input_record, dict):
    record_inputs = input_record.get("inputs")
    if isinstance(record_inputs, dict):
        embedded_image_path = record_inputs.get("input_image_path")
        if isinstance(embedded_image_path, str) and embedded_image_path.strip():
            from pathlib import Path as PathLib
            candidate_path = PathLib(embedded_image_path.strip())
            if candidate_path.exists() and candidate_path.is_file():
                inputs["image_path"] = str(candidate_path.resolve())
                inputs["input_source"] = "image_path_from_embed_record"  # 审计标记
```

- 输入源优先级（完整链）：
  1. cfg.image / cfg.latent（显式配置）
  2. input_record.image / input_record.latent（显式输入）
  3. cfg 解析的 image_path（配置级路径）
  4. **embed_record["inputs"].image_path**（fallback，仅此处新增）

- 每层都显式标记 `input_source`，避免隐式路由歧义

Colab 端缺失输入修复

## 问题诊断

Colab 上运行 paper_full_cuda 完整工作流时，detect 阶段虽然返回码为 0，但生成的 detect_record.json 中：
- `content_evidence_payload.status = "failed"`
- `content_failure_reason = "mask_extraction_no_input"`

导致后续 calibrate 阶段无法继续。

### 根本原因

detect orchestrator 调用 content_extractor 时，inputs 为 None。产生原因：
1. **embed 阶段**：虽然成功执行并生成 embed_record.json，但未保存输入图像路径信息
2. **detect 阶段**：
   - 尝试从 cfg["__detect_final_latents__"] 读取（由 detect CLI 自己的 inference 产生）
   - 尝试从 input_record (embed_record) 读取image/latent字段
   - 尝试解析 image_path
   - 当所有来源都失败时，inputs=None，返回 `mask_extraction_no_input`

## 修复方案

### 修改 1：embed CLI 保存输入信息到 record
**文件**：`main/cli/run_embed.py` 行 505-520
- 在 embed_record 中增加 `inputs` 字段，保存 `input_image_path`
- 供 detect 侧作为回退方案使用

```python
inputs_record = {}
if isinstance(input_image_path, str) and input_image_path.strip():
    inputs_record["input_image_path"] = input_image_path.strip()
if inputs_record:
    record["inputs"] = inputs_record
```

### 修改 2：detect orchestrator 增强输入检索
**文件**：`main/watermarking/detect/orchestrator.py` 行 1230-1310
- 在标准输入来源都失败时，新增回退逻辑
- 从 input_record["inputs"]["input_image_path"] 读取路径
- 如果路径存在，用于掩码提取

```python
# 从 input_record (embed_record) 中读取保存的 input_image_path 作为回退方案
if isinstance(input_record, dict):
    record_inputs = input_record.get("inputs")
    if isinstance(record_inputs, dict):
        embedded_image_path = record_inputs.get("input_image_path")
        if isinstance(embedded_image_path, str) and embedded_image_path.strip():
            from pathlib import Path as PathLib
            candidate_path = PathLib(embedded_image_path.strip())
            if candidate_path.exists() and candidate_path.is_file():
                inputs["image_path"] = str(candidate_path.resolve())
                inputs["input_source"] = "image_path_from_embed_record"
```

### 修改 3：onefile 强制 paper_full_cuda 使用 cuda
**文件**：`scripts/run_onefile_workflow.py` 行 2286-2296
- 在 run_onefile_workflow 中增加 device_override 参数
- 当 profile = paper_full_cuda 且未显式指定 cpu 时，强制设置为 cuda
- 防止 cpu 执行导致 inference 失败

```python
# paper_full_cuda profile requires cuda device
if profile == PROFILE_PAPER_FULL_CUDA and device_override != "cpu":
    print("[onefile] paper_full_cuda profile detected, enforcing device=cuda")
    device_override = "cuda"
```
