# 项目修复方案

> 审计版本号：2026-03-02-0300
> 依据文档：doc/审计/2026-03-02-0300审计报告.md
> 执行目标：解决 Embed 模型加载路径问题，增强 Detect 输入解析安全性，恢复 Sidecar 场景下校准样本纯度。

---

## 1. 修复目标与验收标准

### P0：修复 Embed 端显著性模型加载失败（修正后归因）
- 问题现状：embed_record 显示 saliency_source_model_unavailable，配置中模型路径为 /content/...（Colab 绝对路径），在本地环境不可用。
- 目标：SemanticMaskProvider 在本地环境可加载有效模型权重，或在模型缺失时 fail-fast 并给出明确报错，不走虚构 impl。
- 验收：
  - 存在有效权重时，embed_record.content_result.status == "ok"。
  - 缺失权重时，报错明确为模型路径/加载问题，不产生隐式降级。

### P1：增强 Detect 端输入解析安全性（显式输入，不做路径猜测）
- 问题现状：仅依赖 image_path 解析，图片未落地时容易触发 mask_extraction_no_input。
- 目标：Detect 编排器优先使用显式输入（image / latent），仅在缺失时回退 image_path，不从 inference_params 等隐式字段猜路径。
- 验收：
  - 无物理图片落地但存在上游 latents 时，内容链可执行。
  - 记录中保留 input_source 标记（image_path / image / latent）用于审计。

### P1：恢复校准样本纯度（脚本标记 + 校准器硬过滤）
- 问题现状：Onefile 生成的 synthetic fallback 样本会进入校准输入，污染 NP 阈值。
- 目标：
  1) 脚本端持续标记 synthetic 样本；
  2) 校准入口硬过滤 synthetic 样本。
- 验收：
  - calibration_record.n_samples 不包含 synthetic 样本。
  - 若仅剩 synthetic 样本，应抛无有效样本错误，不输出 0.0 阈值伪结果。

### P2：仅做 Profile 级 Sync 参数调优（避免全局语义漂移）
- 问题现状：直接改全局默认值会影响其他 profile 历史可比性。
- 目标：仅对 paper_full_cuda 生成配置应用 embed.geometry.sync_strength 调优（例如 0.2）。
- 验收：
  - profile_paper_full_cuda.yaml 含该值；
  - 其他 profile 配置不受影响。

## 2. 变更执行计划

### 2.1 [Script] 修复 Embed 模型路径与输入注入
文件：scripts/run_onefile_workflow.py
- 变更动作：
  - 在 paper_full_cuda 配置生成阶段，检测 mask.semantic_model_path：
    - 若为 /content/... 且本地不存在，执行受控路径映射（映射规则固定、可审计）；
    - 映射后仍不存在则 fail-fast，报错并终止。
  - 不引入 dummy_mask_v1 等未注册 impl。

### 2.2 [Code] Detect 编排器支持显式 latent 输入
文件：main/watermarking/detect/orchestrator.py
- 变更动作：
  - 修改 _build_content_inputs_for_detect：
    - 优先收集 inputs.image / inputs.latent（显式来源）；
    - 其次使用 _resolve_detect_image_path；
    - 记录 input_source 字段用于审计。
  - 确保 SemanticMaskProvider 已兼容 inputs["latent"] 路径。

### 2.3 [Code] 校准入口过滤 synthetic 样本（正确模块落点）
文件：main/watermarking/detect/orchestrator.py
- 变更动作：
  - 在 load_scores_for_calibration 遍历 records 时增加过滤：
    - content_evidence_payload.calibration_sample_is_synthetic_fallback == true -> skip；
    - 或 calibration_sample_origin in {"synthetic_fallback", "sidecar_disabled_fallback"} 且 usage 标记为 synthetic -> skip。
  - 将过滤计数写入 strata_info.sampling_policy（如 n_rejected_synthetic_fallback）。

### 2.4 [Script] Profile 级 Sync 参数覆盖（仅 embed.geometry）
文件：scripts/run_onefile_workflow.py
- 变更动作：
  - 在 paper_full_cuda 的 profile config 生成中设置 embed.geometry.sync_strength=0.2（或按实验值）。
  - 不增加 detect.geometry.sync_strength（当前同步实现不读取该字段）。

## 3. 验证步骤

1) 执行单次验证：
```bash
python scripts/run_onefile_workflow.py --profile paper_full_cuda
```

2) 检查 Embed Record：
- 存在模型时：status == "ok"，content_failure_reason 不是 saliency_source_model_unavailable。
- 缺失模型时：流程明确 fail-fast，错误指向 semantic_model_path 不可用。

3) 检查 Detect Record：
- 在无图片场景可见 input_source=latent 且内容链可执行；
- 不再依赖隐式路径推导。

4) 检查 Calibration Record：
- n_samples 排除 synthetic 样本；
- strata_info 含 synthetic 过滤计数；
- 若无真实样本，报无有效样本错误。

5) 回归测试建议：
```bash
python -m pytest tests/test_onefile_workflow_paper_full_profiles.py -q
python -m pytest tests/test_detect_strict_closure.py tests/test_detect_plan_mismatch_is_mismatch.py -q
```

## 4. 回滚计划

- 路径映射逻辑导致误判时，回滚映射并改为显式手动配置 semantic_model_path。
- 若 detect 显式 latent 输入改动影响旧链路，保留 image_path 路径为兼容回退。
- 若 synthetic 过滤导致历史报告差异，保留审计字段并补充版本说明，不关闭过滤规则。
