#### 2.2 main/cli/run_embed.py — Preview Generation 推理逻辑插入

**插入位置：** 现有 `__embed_input_image_path__` 赋值块之后、`content_inputs_pre = embed_orchestrator._build_content_inputs_for_embed(cfg)` 行之前。

**核心逻辑：**
1. 读取 `cfg.embed.preview_generation.enabled`
2. 若 `enabled=True` 且 `cfg["__embed_input_image_path__"]` 仍为 None：调用 `infer_runtime.run_sd3_inference(cfg, pipeline_obj, device, seed, injection_context=None, injection_modifier=None, capture_final_latents=False)`，将输出图像用 `tempfile.mkstemp()` 保存为临时 PNG，路径写入 `cfg["__embed_input_image_path__"]`
3. preview 推理失败时**不中断流程**，保持 `__embed_input_image_path__` 为 None，失败语义传播至内容链（`injection_mode` 降级为 `latent_direct_fallback`）
4. `run_meta["preview_generation"]` 写入审计字段：`{enabled, status, reason, seed, tmp_path}`

**输出变化：** `embed_record.preview_generation` 新增字段（append-only）；若 preview 成功，`embed_record.inputs.input_image_path` 将为临时 PNG 路径（语义等价于真实 SD3 生成图，非合成梯度图）。

**变量作用域说明：** 插入点前 `pipeline_result`、`seed_value` 已赋值；通过 `pipeline_result.get("pipeline_obj")` 和 `cfg.get("device", "cpu")` 取值，不依赖同名局部变量。

#### 2.5 main/cli/run_detect.py — calibration_artifact_path 自动发现逻辑插入

**插入位置：** 现有 `thresholds_path` 注入块（原 lines 461-469）之后。

**核心逻辑：**
```python
if cfg.get("__thresholds_artifact__") is None:
    _detect_cfg_p1a = cfg.get("detect") or {}
    _cal_artifact_path = _detect_cfg_p1a.get("calibration_artifact_path")
    if isinstance(_cal_artifact_path, str) and _cal_artifact_path:
        _cal_resolved = str(Path(_cal_artifact_path).resolve())
        try:
            _thresholds_obj = detect_orchestrator.load_thresholds_artifact_controlled(_cal_resolved)
            cfg["__thresholds_artifact__"] = _thresholds_obj
            run_meta["thresholds_path_injected"] = _cal_resolved
            run_meta["thresholds_source"] = "detect_calibration_artifact_path"
        except Exception as exc:
            set_failure_status(run_meta, RunFailureReason.CONFIG_INVALID, exc)
            raise
```

**优先级逻辑：** `thresholds_path` 参数优先于 `detect.calibration_artifact_path`（参数注入胜配置读取）。加载失败时 fail-fast，不允许降级为 fallback。调用 `load_thresholds_artifact_controlled()`（与现有 thresholds_path 参数使用同一受控加载函数），不旁路 path_policy。

**输出变化：** `detect_record.thresholds_source` 新增值 `"detect_calibration_artifact_path"`（append-only）。

## 3. 实施过程中的问题与修正

### BUG-1：whitelist_version bump 导致 freeze_gate 版本校验失败

**问题：** `runtime_whitelist.yaml` 升至 v1.1 后，`freeze_gate._enforce_policy_semantics_whitelist_version_binding()` 发现 `whitelist_version=v1.1 != policy_path_semantics_version=v1.0`，导致 `GateEnforcementError`，测试 `test_freeze_gate_assert_prewrite_blocks_invalid_record` 失败（错误信息不含 "contract/required/missing"）。

**修复：** 同步将 `configs/policy_path_semantics.yaml` 的 `policy_path_semantics_version` 从 v1.0 升至 v1.1，使两文件版本一致，freeze_gate 版本校验恢复正常。

**教训（RISK-01 兑现）：** 修复方案 RISK-01 已预警此风险，实施时须同步 bump 两个版本号。
