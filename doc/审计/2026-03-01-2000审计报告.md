# 项目风险与研究差距综合审计报告

> 审计版本号：2026-03-01-2000  
> 审计方式：真实静态源码逐行阅读 + 真实 pytest 执行（178 个测试文件）  
> 审计目标：按“静态可确认差距项 / 真实 SD workflow 才能判定项 / 后期消融实验判断项”给出可执行整改结论。

## 静态可确认差距项

### 1) HIGH-CRITICAL：`paper_full_cuda` 强制 `model_v2` 语义掩码但缺失可用权重路径，内容链可确定性失败

**差距项说明**
- 配置将 `mask.saliency_source` 设为 `model_v2`，并启用 `enable_mask=true`。
- 现有实现中，`model_v2` 可用性依赖本地权重路径探测；不可用且未 fallback 时直接失败。

**证据位置**
- 配置：`configs/paper_full_cuda.yaml`（`enable_mask: true`、`mask.saliency_source: model_v2`）
- 代码：`main/watermarking/content_chain/semantic_mask_provider.py`
  - `_probe_model_v2_availability()` 可用性探测
  - `source_selected=model_v2 && model_available=False` 的失败分支（`content_failure_reason=saliency_source_model_unavailable`）
  - `select_saliency_source()` 在显式 `model_v2` 不可用时 `fallback_used=False`

**影响说明**
- 语义显著性驱动的内容证据链无法稳定产出 `ok`，上游失败会主导后续消融与评估，核心论文主张不可闭环验证。

---

### 2) HIGH：`run_onefile_workflow.py` 存在失败语义被改写为可校准分数的可触发路径

**差距项说明**
- 分数恢复分支会从多个候选字段提取数值，一旦恢复成功即将 `content_evidence_payload.status` 改写为 `ok`，并清空失败原因。
- 该行为会改变“失败可拒绝证据链”的原始语义边界。

**证据位置**
- 代码：`scripts/run_onefile_workflow.py`
  - 候选字段集合：`detect_lf_score`、`lf_score`、`score_parts.*`、`record.score` 等
  - 恢复后改写：`status="ok"`、`content_failure_reason=None`
  - 输出：`detect_record_for_scoring.json`
- 备注：同文件存在 paper 模式下“无可恢复字段即拒绝继续”的硬阻断，但未完全覆盖“恢复后改写状态”路径。

**影响说明**
- 失败样本可能被伪装为可校准分数，污染 NP 校准输入与消融解释性。

---

### 3) HIGH：`allow_failed_semantics_collection=true` 放松 detect 门禁，可能在无有效分数样本时继续校准/评估

**差距项说明**
- 当 `valid_content_score_samples` 低于下限且允许 failed semantics collection 时，门禁可返回 relaxed，流程继续。

**证据位置**
- 配置：`configs/paper_full_cuda.yaml`（`experiment_matrix.allow_failed_semantics_collection: true`）
- 代码：`main/evaluation/experiment_matrix.py`（`valid_count < minimum_required` 时的 `gate_relaxed=True` 分支）

**影响说明**
- 可能形成“流程跑完但统计前提不成立”的伪闭环，削弱结果可信度与可比性。

---

### 4) MEDIUM：攻击协议覆盖审计脚本与测试契约不一致（8 个失败用例）

**差距项说明**
- `extract_reported_conditions()` 对输入 item 的 `status` 与 `n_total` 有严格要求；测试构造数据缺少 `status` 时被整体跳过，导致覆盖集为空及后续断言连锁失败。

**证据位置**
- 代码：`scripts/audits/audit_attack_protocol_report_coverage.py`（`extract_reported_conditions()` 过滤条件）
- 测试：`tests/test_audit_attack_protocol_report_coverage.py`（实测 8 个失败用例）

**影响说明**
- 审计链在“覆盖度判定”核心环节与测试契约不一致，存在 sign-off 误判风险。

---

### 5) HIGH-CRITICAL：`np.corrcoef` 路径触发进程级 Aborted，detect 阶段可直接崩溃

**差距项说明**
- 在 `_build_attention_maps_from_latents()` 路径中调用 `np.corrcoef(channels)`，在特定输入维度/运行环境组合下出现进程级 Abort（非 Python 异常）。
- 在 `sync_primary_anchor_secondary=true` 的配置下，该路径具备高触发概率。

**证据位置**
- 代码：`main/watermarking/detect/orchestrator.py`（`np.corrcoef(channels)` 调用点）
- 配置：`configs/paper_full_cuda.yaml`（`sync_primary_anchor_secondary: true`）
- 测试现象：`tests/test_anchor_gate_by_sync_status.py` 执行中进程 Aborted（无 traceback）

**影响说明**
- detect 进程级崩溃会导致记录工件与失败语义无法落盘，双证据链直接失效。

---

### 6) HIGH-CRITICAL：`encode_low_freq_dct` 路径触发进程级 Aborted，embed 阶段可直接崩溃

**差距项说明**
- LF 编码路径在特定输入形状/数值条件下触发底层库 Abort，Python 层不可恢复。

**证据位置**
- 代码：`main/watermarking/content_chain/low_freq_coder.py`（`encode_low_freq_dct()` 调用链）
- 测试现象：`tests/test_c3_real_embedding_pipeline.py` 执行中进程 Aborted（无 traceback）

**影响说明**
- embed 阶段崩溃将阻断整条 workflow，后续 detect/calibrate/evaluate 无法启动。

## 真实 SD workflow 才能判定项

### 1) 几何证据链（同步模板 + 低阶几何反演）在攻击强度梯度下的可用性边界

**需要 workflow 的原因**
- 成功率、失败语义稳定性、以及随攻击强度的退化形态依赖真实模型输出与真实扰动后的数值行为，静态代码无法替代。

### 2) Attention 辅锚点“仅在主同步成功后参与”的执行一致性

**需要 workflow 的原因**
- hook 命中、层名兼容、张量形状与聚合路径均是运行期属性，必须在真实 SD 推理链中验证。

### 3) 内容域 HF/LF + 子空间规划在弱/强攻击下的统计行为

**需要 workflow 的原因**
- 单调性、失败边界、判别力与隐蔽性权衡只能通过真实样本和攻击 sweep 得出，静态结构无法证明。

### 4) NP 校准闭环是否满足目标 FPR 与工件一致性约束

**需要 workflow 的原因**
- 阈值有效性、evaluate 只读阈值约束、ablation 可比性均依赖真实 records + thresholds 联动验证。

## 后期消融实验判断项

### 1) 有利因素（结构层）

- `experiment_matrix.ablation_variants` 已覆盖禁用几何/内容/辅锚点等关键主轴，具备消融编排基础。

### 2) 当前阻断因素

- **进程级阻断**：CRASH-1（`np.corrcoef`）与 CRASH-2（`encode_low_freq_dct`）使主流程无法形成完整工件链。
- **语义级阻断**：上游掩码确定性失败、失败语义改写路径、门禁放松策略叠加，导致结果即使产出也可能不可解释。

### 3) 结论

- 当前状态下，`paper_full_cuda` 消融实验不具备“可运行 + 可解释 + 可比较”三项基本条件。  
- 需先解除进程级阻断，再修复语义与门禁一致性，方可进入实质性消融阶段。

---

## 四、建议修复顺序

### P0（先解阻断）

1. 修复 `encode_low_freq_dct` 进程级崩溃路径（embed 先可运行）。
2. 修复 `np.corrcoef` 进程级崩溃路径（detect 先可运行）。

**目标验收**
- 相关崩溃测试不再 Aborted，改为可记录的 `failed` 语义或成功执行。
- 单条样本可跑通 embed → detect → calibrate → evaluate 基线流程。

### P1（修语义一致性）

3. 修复 `model_v2` 掩码可用性策略（补权重路径或显式降级分组）。

**目标验收**
- `content_failure_reason` 与 `status` 保持语义一致。
- 校准输入不再混入被改写的结构性失败样本。

### P2（修统计与审计门禁）

5. 收紧 `allow_failed_semantics_collection` 使用边界，避免无效样本进入统计闭环。
6. 修复攻击覆盖审计脚本与测试契约不一致问题（8 个失败用例清零）。

**目标验收**
- 校准/评估只在最小有效样本门槛满足时执行。
- `test_audit_attack_protocol_report_coverage.py` 全绿。
