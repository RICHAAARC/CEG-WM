**门控失败根因——条件 1（首要）：**  
`final_latents = cfg.get("__detect_final_latents__")` = `None`。  
此字段由 detect CLI 阶段运行 SD3 推理后注入，但当前 Colab 流程中 detect 未执行独立 SD3 推理（detect 仅读取 watermarked image，不重新运行 SD3 denoiser），故该字段始终为 None。

**门控失败根因——条件 2（即使 latents 可用）：**  
Chain A bug 导致 `mask_digest=null` → embed 侧 `plan_digest` 无效 → detect 侧 `lf_status="absent"` → `detect_lf_status` 无法为 `"ok"`。

**fusion 层失败根因：**

```
NeumanPearsonFusionRule.fuse() [fusion/decision.py]
  → _validate_fusion_inputs(content_evidence, geometry_evidence)
  → content_evidence.status = "failed"
  → content_score = null（_bind_scores_if_ok 未提升）
  → fusion 无法执行 NP 判决
  → decision_status = "error"
```

**产物证据：**
- `detect_record.json:1525`: `"detect_runtime_mode": "fallback_identity_v0"`
- `detect_record.json:1536,1543,1550`: `"decision_status": "error"`（三处：final_decision / fusion_result / content_evidence 各层）
- `detect_record.json:2382`: `"baseline_absent_reason": "detect_runtime_mode_not_real:fallback_identity_v0"`

**修复方案：**  
两阶段：
1. Chain A fix（`parents[3]`）→ `mask_digest` 有效 → `plan_digest` 有效  
2. detect CLI 须支持 SD3 推理 latents 捕获：`cfg["__detect_final_latents__"]` 需由 detect 阶段执行 SD3 encode+denoise 后注入（等同于 embed 阶段的 `__embed_final_latents__`）

**修复后下一个阻断点：**  
即使 `final_latents` 可注入，`detect_lf_status=="ok"` 仍依赖 LF PRC 解码成功；PRC 解码需要 `lf_basis`（来自 SubspacePlanner），而 SubspacePlanner 在 detect 侧通常走 surrogate 路径（`subspace_evidence_level="surrogate"`）。需验证 surrogate 路径下 `detect_lf_status` 是否能达到 `"ok"`。

**链路 C 状态：❌ 未达成（双重阻断：代码 Bug + detect 侧缺少 SD3 latents 注入机制）**

### 链路 D：TPR@FPR 统计指标可计算性

**目标字段：**
- `evaluate_record.tpr_at_fpr_primary` → ✅ **实际：`1.0`（100% TPR）**
- `evaluate_record.fpr_empirical <= target_fpr (0.01)` → ❌ **实际：`0.03333`（3.3% >> 1%）**
- `evaluate_record.evaluation_mode == "real"` → ✅ **实际：`"real"`**
- 计算所用分数 == 论文目标分数（subspace+LF+HF content score）→ ❌ **实际：HF 像素域旁路分数**

**指标追踪：**

```
calibration_record:
  calibration_mode = "real"  ✅
  calibration_is_fallback = false  ✅
  calibration_samples = 30  ✅（满足 minimal_ground_truth_pair_count=30）
  n_selected_null = 30  ✅（NP 框架 FPR 校准使用 null-label 样本）
  threshold_value = -0.006093647556215525  ← 该阈值基于 HF 像素域旁路分数，非论文分数

evaluate_record:
  evaluation_mode = "real"  ✅
  evaluation_is_fallback = false  ✅
  tpr_at_fpr_primary = 1.0  ✅（形式正确，但在错误分数上计算）
  fpr_empirical = 0.03333 = 1/30  ❌（30 null 样本 → 最小可达 FPR = 1/30 = 3.3%，目标 1% 不可达）
  confusion = {fp:1, tn:29, tp:30, fn:0}
```

**FPR 超标根因（统计 + 语义双重）：**

1. **统计根因（样本量不足）：** 30 null 样本下离散 FPR 最小粒度为 1/30 ≈ 3.3%；目标 `target_fpr=0.01`（1%）要求至少 100 null 样本（最优场景 0/100=0%，最小非零 1/100=1%）。NP 校准可以报告 `fpr_empirical=0` 仅在 0 误报时，但无法对外宣称 α<1/30。论文级别声明需 100+ null 样本。

2. **语义根因（分数路径错误）：** 校准与评估使用的 content score 来自 `_extract_content_raw_scores_from_image()`（HF 像素域旁路路径），而非论文声明的 subspace 投影 + LF PRC + HF 融合分数。当前 score = `hf_score_raw = -0.006092...`；论文声明 score 需 Chain A（mask_digest）+ Chain C（final_latents）均修复后方可计算。

**D 链路可计算性结论：**  
数值指标字段均已填充（非 fallback），但指标所指向的分数路径与论文方法不符；FPR 超标不可通过调参修复（需增大样本量 + 修复 Chain A/C）。

**链路 D 状态：⚠ 字段层面可计算，分数语义与论文不符，FPR 超标**
