## 2. 修复项明细

### S1（P0）：Python 3.10 兼容性修复

**问题**
- `from datetime import UTC` 在 Python 3.10 不可用，onefile 入口会 ImportError。

**操作文件**
- `scripts/run_onefile_workflow.py`

**具体修复操作**
```python
# 修复前
from datetime import UTC, datetime

# 修复后
import sys as _sys
from datetime import datetime, timezone as _tz
if _sys.version_info >= (3, 11):
    from datetime import UTC
else:
    UTC = _tz.utc
```

**验证**
- `python -c "import scripts.run_onefile_workflow"`（Python 3.10 / 3.11）

**审计留痕**
- 仅兼容层替换，不触及 gate、audit、records schema。

### S3（P0）：禁止 coverage 预填充绕过

**问题**
- onefile 可写入 `status=absent` 的预填条目，coverage 审计仅看 `group_key` 会误判覆盖达成。

**操作文件 A**
- `scripts/run_onefile_workflow.py`

**具体修复操作 A**
```python
# 函数入口新增 profile 约束
profile = _normalize_profile(profile)
if profile == PROFILE_PAPER_FULL_CUDA:
    return
```
- 函数签名：
  `_ensure_attack_protocol_report_coverage_ready(repo_root, run_root)`
  → `_ensure_attack_protocol_report_coverage_ready(repo_root, run_root, profile)`

**操作文件 B**
- `scripts/audits/audit_attack_protocol_report_coverage.py`

**具体修复操作 B**
```python
for item in metrics_by_condition:
    if not isinstance(item, dict):
        continue
    status = item.get("status")
    if status not in {"ok", "success", "completed"}:
        continue
    if item.get("onefile_pre_audits_fill") is True:
        continue
    if not isinstance(item.get("n_total"), (int, float)):
        continue
    group_key = item.get("group_key")
    if isinstance(group_key, str) and group_key not in conditions:
        conditions.append(group_key)
```

**验证**
- `scripts/audits/audit_attack_protocol_report_coverage.py --run-root <path>`
- 仅满足状态 + 非预填 + `n_total` 数值的条目计入 covered。

**审计留痕**
- paper 覆盖判定从“键覆盖”回到“执行覆盖”。

### S4（P0）：补齐 `grid_summary.json` 计数字段

**问题**
- 缺失 `total/succeeded/failed` 使 onefile 阻断读取默认 0，可能误判。

**操作文件**
- `main/evaluation/experiment_matrix.py`

**具体修复操作**
```python
total_count = len(results)
succeeded_count = sum(
    1 for r in results
    if isinstance(r, dict) and r.get("status") == "ok"
)
failed_count = total_count - succeeded_count

records_io.write_artifact_json_unbound(
    run_root=batch_root,
    artifacts_dir=artifacts_dir,
    path=str(grid_summary_path),
    obj={
        "strict": strict,
        "total": total_count,
        "executed": len(results),
        "succeeded": succeeded_count,
        "failed": failed_count,
        "results": results,
        **anchors_obj,
    },
)
```

**验证**
- 任意 grid 运行后检查 `grid_summary.json` 字段齐全且计数一致。

**审计留痕**
- append-only 增字段，不改既有字段语义。

### S5（P0）：修复 `sync_primary` 下 `relation_digest` 顺序冲突

**问题**
- sync v2 要求 `relation_digest`；`sync_primary` 路径先 sync，未先构建 relation，导致稳定 mismatch。

**操作文件**
- `main/watermarking/detect/orchestrator.py`

**具体修复操作**
```python
if sync_primary_mode:
    relation_digest_seed = _precompute_relation_digest_for_sync(
        cfg,
        enable_attention_proxy=enable_attention_proxy,
    )
    if enable_sync:
        sync_base_inputs = _build_geometry_runtime_inputs(
            cfg,
            enable_attention_proxy=enable_attention_proxy,
        )
        if isinstance(relation_digest_seed, str) and relation_digest_seed:
            sync_base_inputs["relation_digest"] = relation_digest_seed
        sync_result = _run_sync_module_for_detect(sync_module, cfg, sync_base_inputs)
```

**必须保持不变的语义**
- `anchor_gated_by_sync_failure`、`anchor_status=absent`、hard gate 规则不改。

**新增函数约束**
- `_precompute_relation_digest_for_sync()` 仅做 runtime 计算，不写 records，不新增 override 路径。

**验证**
- mock 测试验证 `sync_primary` 时 sync inputs 获得 relation digest；sync 失败仍保持原 gate 语义。
