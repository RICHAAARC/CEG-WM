### P0-5 修复 sync_primary 模式下 relation_digest 顺序冲突（S5）

**问题**：`sync_primary` 模式先调 sync，但 sync v2（`geometry_latent_sync_sd3_v2`）要求 `relation_digest` 来自 anchor 的注意力图对比结果，此时 anchor 尚未运行，导致稳定 `mismatch`。

**根因**：`relation_digest` 在架构上属于 anchor 输出，而 sync_primary 语义要求 sync 先于 anchor——两者形成循环依赖。

**修复策略（最小改动，不改变 v2 合约，且保持现有 hard-gate 语义）**：  
不改为“先 anchor 再 sync”。保持现有 `sync_primary` 执行语义（sync 先、sync 失败时 anchor 仍为 `absent`），仅新增“sync 前 relation_digest 预计算注入”：

1. 在 `sync_primary` 分支进入 sync 前，新增 `_precompute_relation_digest_for_sync(...)`；  
2. 该函数只基于 runtime self-attention（或允许的 attention 输入）计算 relation 摘要，不产出/写入 anchor 证据字段；  
3. 将预计算得到的 `relation_digest` 写入 sync runtime_inputs，再执行现有 sync；  
4. anchor 执行与 `anchor_gated_by_sync_failure` 行为保持不变（不改变现有审计与测试语义）。

**修复文件**：`main/watermarking/detect/orchestrator.py`

改动 `_run_geometry_chain_with_sync()` 中 `sync_primary_mode` 分支的 sync 输入构造块：

```python
if sync_primary_mode:
    # 预计算 relation_digest，仅供 sync_primary 的 sync 输入使用，不改 anchor 语义
    relation_digest_seed = _precompute_relation_digest_for_sync(
        cfg,
        enable_attention_proxy=enable_attention_proxy,
    )
    if enable_sync:
        sync_base_inputs = _build_geometry_runtime_inputs(
            cfg,
            enable_attention_proxy=enable_attention_proxy,
        )
        if isinstance(relation_digest_seed, str) and relation_digest_seed:
            sync_base_inputs["relation_digest"] = relation_digest_seed
        sync_module = getattr(impl_set, "sync_module", None)
        sync_result: Dict[str, Any] = _run_sync_module_for_detect(
            sync_module, cfg, sync_base_inputs
        )
```

**重要约束**：  
- `sync_primary` 下既有 `anchor_gated_by_sync_failure`、`anchor_status=absent`、`hard_gate` 行为必须保持不变（避免引入语义回归）。  
- 不修改 v2 sync 模块 `latent_sync_template.py`，不修改 `frozen_contracts.yaml`，不修改 whitelist。  
- 新增预计算函数仅作为 runtime 输入准备，不写 records、不新增 override 路径。

**实现缺口（实施时必须补充）**：`_precompute_relation_digest_for_sync()` 在 `orchestrator.py` 中**尚不存在**，需新增。该函数应：  
1. 调用 `_resolve_runtime_self_attention_maps(cfg)` 获取当前可用 attention 输入；  
2. 如有 attention 输入，以其 shape/mean/std 计算确定性 digest（参考 `_build_geometry_runtime_inputs` 中已有的 `digests.canonical_sha256` 用法）；  
3. 如无 attention 输入，返回 `None`（sync v2 收到 `None` 仍会 mismatch，但这是正确的失败语义，不是新引入的问题）；  
4. 不写任何 artifact，不修改 cfg，纯计算返回值。
