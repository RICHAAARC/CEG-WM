## S2：SD3 Self-Attention Map 提取空壳

### 2.1 当前问题

`_extract_runtime_self_attention_maps()`（infer_runtime.py L418）会探测 output / pipeline 上多个属性名（`self_attention_maps`, `attention_maps`, `runtime_self_attention_maps` 等），但缺少稳定生产该字段的 runtime capture 机制，实际上始终返回 `(None, "<absent>")`。detect 链已支持通过 `__runtime_self_attention_maps__` 注入（orchestrator.py `_resolve_runtime_self_attention_maps`），但没有 hook 能产出该字段。hooks.py 只有注入回调，无 `register_forward_hook` 抓取 attention。几何锚点全链路为 latent correlation proxy（`anchor_evidence_level="proxy"`）。

### 2.2 修复目标

- 在 SD3 Transformer 的指定 attention layer 上注册 `register_forward_hook`，捕获 self-attention map tensor
- 将捕获的 attention maps 传递给 `AttentionAnchorMapRelation.extract()` 用于真实几何锚点
- `anchor_evidence_level` 从 `"proxy"` 提升到 `"authentic"`（代码中 `authentic` 表示真实 attention 数据，区别于 `proxy`）

### 2.3 需要新增/修改的函数

| 函数 | 操作 | 文件 |
|------|------|------|
| `class SelfAttentionCaptureHook` | **新增** | hooks.py |
| `register_attention_hooks(pipeline, cfg) → SelfAttentionCaptureHook` | **新增** | hooks.py |
| `remove_attention_hooks(hook_handle) → None` | **新增** | hooks.py |
| `_extract_runtime_self_attention_maps(...)` | **修改** | infer_runtime.py L418 — 优先从 hook 缓存读取 |
| `run_sd3_inference(...)` | **修改** | infer_runtime.py L28 — 新增 `attention_capture` 参数 |
| `AttentionAnchorMapRelation.extract(...)` | **修改** | attention_anchor_extractor.py — 检测 `anchor_evidence_level` 并切换路径 |

### 2.4 实施步骤

**(1)** hooks.py 新增 `SelfAttentionCaptureHook` 类：

> **重要技术说明**：diffusers 的 `Attention` 模块（`AttnProcessor2_0` 等）内部调用 `F.scaled_dot_product_attention`，其 `forward()` 仅返回 `hidden_states`（单 tensor），**不返回 attention weights**。因此不能用简单的 `register_forward_hook` 捕获 output 来获取 attention map。
>
> **正确方案**：在 Q/K 线性投影层（`to_q`, `to_k`）上注册 hook 捕获投影输出，然后在推理结束后外部计算 `softmax(Q @ K^T / sqrt(d))` 得到 attention weights。或者替换 attention processor 为自定义版本，在内部存储 attention weights。

```python
class SelfAttentionCaptureHook:
    """
    功能：捕获 SD3 Transformer self-attention Q/K 投影，按模块名+step 维度缓存配对，事后计算 attention weights。

    Capture Q/K projections from SD3 Transformer Attention layers,
    keyed by (module_name, step_index) to avoid cross-layer/cross-step mismatch.
    Only captures self-attention (.attn1) layers; cross-attention (.attn2) is excluded.
    Supports optional layer sampling to limit memory and latency.

    Args:
        max_layers: Maximum number of attention layers to hook (None = all self-attn layers).
        sample_steps: Set of denoising step indices to capture (None = all steps).
    """
    def __init__(self, max_layers=None, sample_steps=None):
        self._captures = {}        # Dict[(module_name, step_idx), {"q": Tensor, "k": Tensor}]
        self._hook_handles = []    # List[RemovableHandle]
        self._step_index = 0       # 当前 denoising step 索引
        self._max_layers = max_layers
        self._sample_steps = sample_steps  # 可选：仅采集指定 step

    def _make_hook_fn(self, module_name, projection):
        """生成闭包 hook，绑定 module_name 和 projection 类型。"""
        def hook_fn(module, input, output):
            # 可选：仅在指定 step 采集
            if self._sample_steps is not None and self._step_index not in self._sample_steps:
                return
            key = (module_name, self._step_index)
            if key not in self._captures:
                self._captures[key] = {}
            self._captures[key][projection] = output.detach().cpu()
        return hook_fn

    def advance_step(self):
        """推进 denoising step 计数器（在每个 scheduler step 后调用）。"""
        self._step_index += 1

    def register(self, pipeline, cfg):
        """
        在 transformer 的 self-attention 子模块的 to_q / to_k 上注册 hook。
        仅针对 self-attention（通常 '.attn1.' 或 '.attn.' 且不含 '.attn2.'），
        跳过 cross-attention（'.attn2.'）以避免拿到错误的 attention map。
        """
        transformer = getattr(pipeline, "transformer", None)
        if transformer is None:
            return
        # SD3Transformer2DModel → transformer_blocks[*].attn.to_q / .to_k
        # 注意：Self-attn 通常在 .attn 或 .attn1，cross-attn 在 .attn2
        hooked_layers = 0
        for name, module in transformer.named_modules():
            # 跳过 cross-attention 层
            if ".attn2." in name:
                continue
            # 限制采样层数
            if self._max_layers is not None and hooked_layers >= self._max_layers:
                break
            if name.endswith(".to_q"):
                # 提取父层名（如 transformer_blocks.0.attn）作为 module_name
                parent_name = name.rsplit(".", 1)[0]
                handle = module.register_forward_hook(self._make_hook_fn(parent_name, "q"))
                self._hook_handles.append(handle)
            elif name.endswith(".to_k"):
                parent_name = name.rsplit(".", 1)[0]
                handle = module.register_forward_hook(self._make_hook_fn(parent_name, "k"))
                self._hook_handles.append(handle)
                hooked_layers += 1  # 按 K hook 计数（一对 Q/K = 一层）

    def compute_attention_maps(self):
        """从收集的 (Q, K) 对计算 attention weights。仅处理 Q/K 都存在的条目。"""
        import torch
        maps = []
        for key in sorted(self._captures.keys()):
            pair = self._captures[key]
            q, k = pair.get("q"), pair.get("k")
            if q is None or k is None:
                # Q/K 不完整，跳过（不会产出错误配对）
                continue
            d_k = q.shape[-1]
            attn_weights = torch.softmax(q @ k.transpose(-2, -1) / (d_k ** 0.5), dim=-1)
            maps.append({"module": key[0], "step": key[1], "attn": attn_weights})
        return maps

    def collect(self):
        """收集 attention maps 并清空缓存。"""
        maps = self.compute_attention_maps()
        self._captures.clear()
        self._step_index = 0
        return maps

    def remove(self):
        """移除所有 hook handles。"""
        for h in self._hook_handles:
            h.remove()
        self._hook_handles.clear()
```

> **关键设计决策说明**：
> - **按 (module_name, step_index) 维度缓存**：避免多层/多 step 下全局 `_current_q/_current_k` 串层配对错误。
> - **跳过 `.attn2.`（cross-attention）**：仅捕获 self-attention，避免拿到 text-image cross-attention map。
> - **`max_layers` 限制采样层数**：SD3 有 24 层 transformer block，全采集内存和时延开销极大；建议默认只采集 4–8 层。
> - **`sample_steps` 限制采集 step**：50 step 推理中可仅采集中间段（如 {10,20,30}）降低开销。
> - **`advance_step()` 必须在 on_step_end 回调中调用**，以保持 step 计数同步。
>
> **替代方案**：若项目后续选择替换 AttnProcessor，可实现 `CaptureAttnProcessor` 在 `__call__` 中调用 `torch.nn.functional.scaled_dot_product_attention` 的同时保存 attention weights 到外部列表。两种方案的上层接口（`register` / `collect` / `remove`）保持一致。
>
> **R1 运行时验证必须确认**：SD3Transformer2DModel 的 `named_modules()` 中 self-attention 与 cross-attention 的确切命名模式（如 `.attn.to_q` vs `.attn2.to_q`），Q/K tensor shape 是否含 multi-head 维度，以及合理的 `max_layers` / `sample_steps` 默认值。

## S3：ECC/LDPC/BP 解码未实现

```python
    """
    功能：在 latent 空间注入 FFT 域同步模板。

    Inject sync template into latent frequency domain for geometric sync.

    Args:
        latents: Input latent tensor/array, shape [B, C, H, W] or [C, H, W].
        cfg: Configuration mapping.
        seed: Deterministic seed for template generation.

    Returns:
        Tuple of (modified_latents, inject_trace).

    Raises:
        TypeError: If inputs are invalid.
        ValueError: If latent shape is unsupported.
    """

        return latents, {"status": "absent", "reason": "embed_sync_disabled"}
    
    # (2) 构建同步模板（FFT 域）
    template = self._build_sync_template(latents.shape, cfg, seed)
    
    # (3) 将 latents 变换到频率域

    latents_fft += template * sync_strength

    modified_latents = np.real(np.fft.ifft2(latents_fft))
    
    # (6) 构建 trace
    inject_trace = {
        "status": "ok",
        "sync_inject_strength": sync_strength,
        "template_digest": digests.canonical_sha256({"shape": list(template.shape), "seed": seed}),
        "sync_config_digest": digests.canonical_sha256(self._build_sync_config_domain(cfg)),
    }
    return modified_latents, inject_trace
```

**(2)** 新增 `_build_sync_template()`：

```python
def _build_sync_template(self, shape, cfg, seed):
    """
    功能：生成确定性 FFT 域同步模板。

    Generate deterministic sync template in FFT domain.

    Args:
        shape: Target latent shape.
        cfg: Config for bin counts.
        seed: RNG seed.

    Returns:
        Complex template array, same spatial shape as latent.
    """
    # 使用固定 seed 在特定频率 bin 放置标记信号
    # bin 选择：sync_fft_bins（默认 16）个位置
    # 每个 bin 的相位由 seed 确定 → detect 侧用相同 seed 重建匹配
    fft_bins = self._resolve_fft_bins(cfg)
    rng = np.random.RandomState(seed)
    template = np.zeros(shape[-2:], dtype=complex)
    # 在对称位置放置已知相位信号
    for _ in range(fft_bins):
        row = rng.randint(1, shape[-2] // 2)
        col = rng.randint(1, shape[-1] // 2)
        phase = rng.uniform(0, 2 * np.pi)
        template[row, col] = np.exp(1j * phase)
        template[-row, -col] = np.exp(-1j * phase)  # 共轭对称
    return template
```

5. S2 依赖 SD3 transformer 的实际 attention layer 命名，建议 R1 运行验证后再固化 hook 层名
